# Промт для создания Telegram-бота «Визитка мастера»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Визитка мастера».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для частных мастеров сферы красоты (маникюр, брови, парикмахерские услуги, визаж). Цель — заменить ссылку на соцсети и текстовый файл с реквизитами единой удобной витриной в Telegram.

**Монетизация:** Продажа готового решения мастерам за 2000–4000 рублей, либо ежемесячная подписка 300 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение с именем мастера (задаётся в переменной), фотографией профиля (загружается из папки проекта) и кратким описанием услуг.
- Под сообщением — клавиатура с кнопками (ReplyKeyboardMarkup или InlineKeyboardMarkup — выбери более подходящий вариант с обоснованием):
  - 📋 Прайс-лист
  - 📸 Мои работы
  - 📍 Контакты и адрес
  - 📞 Записаться
  - 📣 Акции (опционально)

### 2. Обработка кнопок

#### Прайс-лист
- Бот отправляет сообщение с текстовым списком услуг и ценами.
- Данные должны браться из внешнего файла (например, `prices.json`) для лёгкого редактирования без изменения кода.

#### Мои работы
- Бот отправляет медиа-группу (альбом) из 5–10 фотографий работ.
- Фотографии хранятся в папке `portfolio/`.
- При отсутствии фотографий выводит сообщение «Фото временно недоступны».

#### Контакты и адрес
- Отправляет текстовое сообщение с номером телефона, ссылкой на Instagram/WhatsApp и геолокацией (широта/долгота задаются в конфиге).
- Добавляет кнопку «Проложить маршрут» с ссылкой на Яндекс.Карты или Google Maps.

#### Записаться
- Бот предоставляет кнопку-ссылку на внешний сервис записи (например, YClients, DIKIDI или просто ссылку на личные сообщения мастера).
- URL задаётся в конфиге.

#### Акции
- Если в конфиге указана активная акция (флаг `promo_active = True`), то отправляет специальный текст и промокод.
- Если акции нет — кнопка скрыта.

### 3. Администрирование (простое, без веб-панели)

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — показывает статистику: количество уникальных пользователей, запустивших бота.
- `/broadcast [текст]` — позволяет сделать рассылку всем подписчикам бота (с подтверждением и задержкой, чтобы избежать блокировок Telegram).

### 4. Технические детали

**Стек:**
- Python 3.10+
- aiogram 3.x
- python-dotenv для конфигурации
- aiofiles для работы с файлами
- asyncio

**Структура проекта:**
```
/bot
├── main.py                    # точка входа
├── config.py                  # загрузка настроек из .env и JSON
├── handlers/
│   ├── user.py                # обработчики пользовательских команд
│   └── admin.py               # обработчики админских команд
├── keyboards/
│   └── main_menu.py           # клавиатуры
├── data/
│   ├── prices.json            # прайс-лист
│   ├── portfolio/             # фото работ
│   └── profile.jpg            # аватар мастера
├── .env                       # токен бота, ID админа, координаты и т.д.
├── requirements.txt

```

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `last_name` TEXT
- `last_seen` TIMESTAMP

При каждом `/start` добавлять или обновлять запись.

### 6. Обработка ошибок и безопасность

- Все внешние запросы должны иметь таймауты и обработку исключений.
- При ошибке чтения файлов или БД бот должен отправлять пользователю вежливое сообщение «Сервис временно недоступен, попробуйте позже», а админу — детали ошибки в ЛС.
- Рассылка должна иметь искусственную задержку 0.05 сек между сообщениями и пропускать пользователей, заблокировавших бота.

### 7. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
      - ./portfolio:/app/portfolio
```

**Инструкция по развёртыванию на VPS:**

1. Скопировать файлы проекта на сервер
2. Создать файл `.env` с необходимыми переменными
3. Заполнить `prices.json` актуальным прайс-листом
4. Положить фото в папку `portfolio/`
5. Запустить через Docker: `docker-compose up -d`
