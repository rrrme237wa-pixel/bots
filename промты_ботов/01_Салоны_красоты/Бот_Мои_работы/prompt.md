# Промт для создания Telegram-бота «Мои работы» (Портфолио мастера)

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Мои работы».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для мастеров сферы красоты (маникюр, тату, парикмахеры, визажисты). Цель — предоставить интерактивное портфолио работ с возможностью фильтрации по категориям, чтобы клиенты могли быстро найти примеры нужных услуг и записаться.

**Монетизация:** Продажа готового решения мастерам за 3000–5000 рублей, либо ежемесячная подписка 500 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение с именем мастера и кратким описанием.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - 🖼️ Смотреть работы
  - 📁 По категориям
  - 🔍 Поиск по хештегам
  - 📞 Записаться
  - ℹ️ О мастере

### 2. Обработка кнопок

#### Смотреть работы
- Бот показывает ленту последних работ (медиагруппы по 5–10 фото).
- Навигация: кнопки «◀️ Назад» / «▶️ Вперёд» для переключения между альбомами.
- Под каждым альбомом — кнопка «❤️ Хочу так же», которая пересылает запрос мастеру.

#### По категориям
- Бот показывает список категорий (например: «Френч», «Однотонное покрытие», «Дизайн», «Наращивание»).
- При выборе категории — показ работ из этой категории.
- Каждая работа сопровождается описанием и временем выполнения.

#### Поиск по хештегам
- Клиент вводит хештег или выбирает из популярных (#френч, #нюд, #яркийдизайн).
- Бот ищет работы с matching хештегами в описании.
- Возвращает найденные работы в виде медиагрупп.

#### Записаться
- Кнопка-ссылка на внешний сервис записи (YClients, DIKIDI) или на личные сообщения мастера.
- URL задаётся в конфиге.

#### О мастере
- Текстовое сообщение с опытом, сертификатами, адресом.
- Фото рабочего места (1–3 фото).

### 3. Администрирование (для мастера)

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — статистика: количество просмотров, популярные категории, количество переходов на запись.

- `/add_work` — добавить новую работу через бота:
  1. Загрузка фото (1–10 шт.)
  2. Выбор категории
  3. Ввод описания
  4. Добавление хештегов

- `/delete_work [id]` — удалить работу из портфолио.

- `/broadcast [текст]` — рассылка подписчикам.

- `/stats` — детальная статистика по каждой работе (просмотры, лайки).

### 4. Технические детали

**Стек:**
- Python 3.10+
- aiogram 3.x
- python-dotenv для конфигурации
- aiosqlite для хранения данных
- aiofiles для работы с файлами
- asyncio

**Структура проекта:**
```
/bot
├── main.py                    # точка входа
├── config.py                  # загрузка настроек из .env
├── handlers/
│   ├── user.py                # обработчики пользовательских команд
│   ├── admin.py               # обработчики админских команд
│   └── portfolio.py           # логика портфолио
├── keyboards/

│   └── portfolio_kb.py        # клавиатуры портфолио
├── utils/

│   └── media_handler.py       # обработка медиа
├── data/
```

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `last_seen` TIMESTAMP

Таблица `works`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `title` TEXT
- `description` TEXT
- `category` TEXT
- `hashtags` TEXT (JSON массив)
- `created_at` TIMESTAMP
- `views` INTEGER DEFAULT 0
- `is_active` BOOLEAN DEFAULT 1

Таблица `work_photos`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `work_id` INTEGER
- `file_id` TEXT
- `file_path` TEXT
- `order` INTEGER

Таблица `categories`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `name` TEXT
- `slug` TEXT UNIQUE

При каждом `/start` обновлять `last_seen` в `users`.

### 6. Обработка ошибок и безопасность

- Таймауты на внешние запросы (10 сек).
- При ошибке чтения файлов — вежливое сообщение пользователю.
- Рассылка с задержкой 0.05 сек.
- Обработка `TelegramForbiddenError`.

### 7. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
```

**Инструкция по развёртыванию:**
1. Скопировать файлы на сервер
2. Создать `.env`
3. `docker-compose up -d`
