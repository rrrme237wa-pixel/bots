# Промт для создания Telegram-бота «Подготовка к приему»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Подготовка к приему».

**Контекст проекта:**
Этот бот продаётся частным клиникам, косметологам и медицинским центрам. Цель — автоматизировать процесс информирования пациентов о подготовке к процедурам (анализы, УЗИ, эндоскопия и т.д.), чтобы снизить количество отмен из-за неправильной подготовки и сэкономить время администраторов на телефонных звонках.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение с названием клиники и кратким описанием возможностей.
   - Под сообщением — клавиатура с кнопками (InlineKeyboardMarkup):
     - 📋 Выбрать процедуру
     - 📅 Мои записи
     - ℹ️ О клинике
     - 📞 Контакты

2. **Обработка кнопок:**
   - **Выбрать процедуру:** Бот показывает список процедур из файла `procedures.json`. Каждая процедура — это кнопка. При выборе бот отправляет подробную памятку (текст + опционально фото/файл).
     - Пример: «Гастроскопия» → «Не есть 8 часов, не пить за 2 часа, взять с собой паспорт и полис».
   - **Мои записи:** Если пациент записан (админ добавил его через команду `/add_patient`), бот показывает дату, время и процедуру, а также кнопку «Получить памятку».
   - **О клинике:** Текст с описанием, адресом, режимом работы. Данные из конфига.
   - **Контакты:** Телефон, ссылка на сайт, кнопка «Позвонить» (tel:), геолокация.

3. **Администрирование:**
   - Команда `/add_patient [user_id] [процедура] [дата] [время]` — добавить пациента в базу. Бот сохранит запись и напомнит пациенту за 24 часа до приема с памяткой.
   - Команда `/list_patients` — показать всех записанных пациентов на ближайшие 7 дней.
   - Команда `/broadcast [текст]` — рассылка всем пациентам (с задержкой 0.05 сек).
   - Команда `/stats` — статистика: всего пациентов, записей на сегодня/неделю.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv, aiosqlite, asyncio.
   - **Структура проекта:**
```
/bot
├── main.py
├── config.py
├── handlers/
│   ├── user.py
│   └── admin.py
├── keyboards/
│   └── main_menu.py
├── data/
│   ├── procedures.json
│   └── clinic_info.json
├── database.py
├── .env
└── requirements.txt
```
   - **Конфигурация (.env):**

   - **Хранение данных:** SQLite с таблицами:
     - `patients` (user_id, username, procedure, appointment_date, appointment_time, created_at)
     - `users` (user_id, username, first_name, last_seen)

5. **Обработка ошибок и безопасность:**
   - Все запросы к БД с обработкой исключений.
   - При ошибке — пользователю вежливое сообщение, админу — лог в ЛС.
   - Рассылка с проверкой на блокировку бота.

6. **Деплой:**

   - Включить Dockerfile, docker-compose.yml, README.md с инструкцией.
