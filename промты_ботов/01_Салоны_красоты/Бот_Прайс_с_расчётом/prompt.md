# Промт для создания Telegram-бота «Прайс с расчётом стоимости»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Прайс с расчётом стоимости».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для частных мастеров сферы красоты (маникюр, бровисты, парикмахеры, косметологи). Цель — предоставить клиентам удобный калькулятор стоимости услуг, где они могут выбрать несколько позиций и увидеть итоговую сумму. Это повышает прозрачность ценообразования и снижает количество уточняющих вопросов мастеру.

**Монетизация:** Продажа готового решения мастерам за 3000–5000 рублей, либо ежемесячная подписка 500 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение с именем мастера (задаётся в переменной) и кратким описанием.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - 🧮 Рассчитать стоимость
  - 📋 Полный прайс-лист
  - 📞 Записаться
  - ℹ️ О мастере

### 2. Обработка кнопок

#### Рассчитать стоимость
- Бот показывает категории услуг (например: «Маникюр», «Педикюр», «Дизайн», «Уход»).
- При выборе категории отображается список услуг с ценами и кнопками [+]/[-] для добавления/удаления из корзины.
- Кнопка «✅ Готово» — показывает итоговую сумму и выбранные услуги.
- Кнопка «🔄 Начать заново» — очищает корзину.
- Кнопка «📞 Записаться с этим набором» — отправляет мастеру сообщение с выбранными услугами и контактом клиента.

#### Полный прайс-лист
- Бот отправляет сообщение с текстовым списком всех услуг и ценами.
- Данные берутся из внешнего файла `prices.json` для лёгкого редактирования без изменения кода.

#### Записаться
- Бот предоставляет кнопку-ссылку на внешний сервис записи (YClients, DIKIDI) или ссылку на личные сообщения мастера.
- URL задаётся в конфиге.

#### О мастере
- Текстовое сообщение с опытом работы, сертификатами, адресом салона.
- Фотографии работ (медиагруппа из папки `portfolio/`).

### 3. Администрирование

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — показывает статистику: количество уникальных пользователей, количество расчётов стоимости, популярные услуги.
- `/broadcast [текст]` — рассылка всем подписчикам (с задержкой 0.05 сек между сообщениями).
- `/export` — выгрузка всех расчётов за период в CSV-файл.

### 4. Технические детали

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `last_name` TEXT
- `first_seen` TIMESTAMP
- `last_seen` TIMESTAMP

Таблица `calculations`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER
- `services` TEXT (JSON массив выбранных услуг)
- `total_price` INTEGER
- `created_at` TIMESTAMP

Таблица `subscriptions`:
- `user_id` INTEGER PRIMARY KEY
- `subscribed_at` TIMESTAMP

При каждом `/start` добавлять или обновлять запись в `users`.

### 6. Обработка ошибок и безопасность

- Все внешние запросы должны иметь таймауты (10 сек) и обработку исключений.
- При ошибке чтения файлов или БД бот должен отправлять пользователю вежливое сообщение «Сервис временно недоступен, попробуйте позже», а админу — детали ошибки в ЛС.
- Рассылка должна иметь искусственную задержку 0.05 сек между сообщениями и пропускать пользователей, заблокировавших бота (обработка `TelegramForbiddenError`).
- Валидация входных данных в калькуляторе (защита от отрицательных значений).

### 7. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
      - ./portfolio:/app/portfolio
```

**Инструкция по развёртыванию на VPS:**

1. Скопировать файлы проекта на сервер
2. Создать файл `.env` с необходимыми переменными
3. Заполнить `prices.json` актуальным прайс-листом
4. Запустить через Docker: `docker-compose up -d`
5. Или через systemd: создать сервис `/etc/systemd/system/beauty-bot.service`

```ini
[Unit]
Description=Beauty Price Calculator Bot
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/beauty-bot
ExecStart=/usr/bin/python3 main.py
Restart=always

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable beauty-bot
sudo systemctl start beauty-bot
```
