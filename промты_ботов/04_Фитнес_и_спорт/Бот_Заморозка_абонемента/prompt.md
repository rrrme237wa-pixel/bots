# Промт для создания Telegram-бота «Заморозка абонемента»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Заморозка абонемента».

**Контекст проекта:**
Этот бот продаётся фитнес-клубам, студиям и спортивным центрам. Цель — автоматизировать процесс заморозки абонементов: клиент подаёт заявку через бота, администратор подтверждает, бот пересчитывает срок действия. Это снижает нагрузку на ресепшн и улучшает клиентский опыт.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение.
   - Под сообщением — клавиатура (InlineKeyboardMarkup):
     - ❄️ Заморозить абонемент
     - 📅 Статус заморозки
     - 📋 Мои абонементы
     - 📞 Контакты

2. **Обработка кнопок и сценарии:**
   - **Заморозить абонемент:** Клиент выбирает абонемент, указывает причину и даты заморозки. Бот отправляет заявку админу.
   - **Статус заморозки:** Показывает статус заявки («На рассмотрении», «Подтверждено», «Отклонено»).
   - **Мои абонементы:** Список активных абонементов с датами.
   - **Контакты:** Телефон, адрес.

3. **Администрирование:**
   - Команда `/list_freeze_requests` — показать все заявки на заморозку.
   - Команда `/approve_freeze [request_id] [days]` — подтвердить заморозку.
   - Команда `/reject_freeze [request_id] [reason]` — отклонить заявку.
   - Команда `/broadcast [текст]` — рассылка.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv, aiosqlite, asyncio.
   - **Структура проекта:**
```
/bot
├── main.py
├── config.py
├── handlers/
│   ├── client.py
│   └── admin.py
├── keyboards/
│   └── main_menu.py
├── database.py
├── .env
└── requirements.txt
```
   - **Конфигурация (.env):**
```
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
CLUB_NAME=FitClub
CLUB_PHONE=+79001234567
MAX_FREEZE_DAYS=30
```
   - **Хранение данных:** SQLite с таблицами:
     - `subscriptions` (id, user_id, type, start_date, end_date, is_frozen, freeze_days_left)
     - `freeze_requests` (id, user_id, subscription_id, reason, start_date, end_date, status, admin_comment)
     - `users` (user_id, username, first_name, last_seen)

5. **Обработка ошибок и безопасность:**
   - Проверка остатка дней заморозки.
   - Уведомление клиента о решении админа.
   - Логирование всех заявок.

6. **Деплой:**
   - Код готов к запуску через Docker или systemd.
   - Включить Dockerfile, docker-compose.yml, README.md.
