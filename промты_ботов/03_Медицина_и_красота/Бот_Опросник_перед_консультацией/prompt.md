Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного Telegram-бота «Опросник перед консультацией».

**Контекст проекта:**
Бот для врачей и клиник. Перед приемом пациент отвечает на серию вопросов (жалобы, история болезни, аллергии), врач получает структурированную анкету до начала консультации. Экономит время приема, улучшает качество сбора анамнеза. Монетизация: продажа клиникам.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При `/start` бот приветствует пациента и предлагает заполнить анкету.
   - Меню пациента:
     - 📝 Заполнить анкету
     - 📋 Мои ответы
     - ❓ Контакты клиники
   - Меню врача:
     - 👥 Анкеты пациентов
     - 📊 Статистика

2. **Обработка кнопок:**
   - **Заполнить анкету:**
     - Бот задает вопросы по очереди (машина состояний FSM):
       1. ФИО
       2. Дата рождения
       3. Основная жалоба (текст)
       4. Хронические заболевания (текст)
       5. Аллергии (текст)
       6. Принимаемые лекарства (текст)
       7. Оценка боли от 1 до 10 (кнопки)
     - После заполнения бот формирует сводку и отправляет врачу.
   - **Мои ответы:** Пациент видит историю заполненных анкет.
   - **Анкеты пациентов (врач):** Список анкет с возможностью фильтрации по дате.

3. **Администрирование (врач):**
   - Команда `/export [user_id]` — выгрузка анкеты пациента в текстовом формате.
   - Настройка списка вопросов через JSON-файл.
   - Статистика: количество заполненных анкет за период.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, sqlite3, python-dotenv, aiofiles.
   - **Структура проекта:**
/bot
├── main.py
├── config.py
├── handlers/
│ ├── patient.py
│ └── doctor.py
├── keyboards/
│ └── menus.py
├── data/
│ └── questions.json
├── .env
└── requirements.txt
   - **Конфигурация (.env):**
BOT_TOKEN=...
DOCTOR_IDS=123456789
CLINIC_NAME="Клиника Здоровья"
CLINIC_PHONE="+79990000000"

5. **Хранение данных:**
   - SQLite таблицы:
     - `users` (user_id, role, first_name, username)
     - `questionnaires` (id, user_id, answers_json, created_at, viewed_by_doctor)

6. **Обработка ошибок и безопасность:**
   - Валидация ввода (проверка даты рождения, обязательные поля).
   - Возможность сохранить черновик анкеты и продолжить позже.
   - Конфиденциальность данных (доступ только у врача).

7. **Деплой:**
   - Dockerfile, docker-compose.yml.
   - Инструкция по настройке.

**Формат ответа:**
Предоставь полный код всех файлов проекта. В конце дай инструкцию по развёртыванию.
