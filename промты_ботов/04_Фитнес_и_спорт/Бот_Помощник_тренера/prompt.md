# Промт для создания Telegram-бота «Помощник тренера»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Помощник тренера».

**Контекст проекта:**
Этот бот продаётся персональным тренерам и фитнес-инструкторам. Цель — автоматизировать выдачу программ тренировок клиентам: тренер загружает программу, клиент получает её частями и отмечает выполнение. Бот повышает вовлечённость и упрощает работу тренера.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение.
   - Для клиента:
     - 🏋️ Моя программа
     - ✅ Отметить выполнение
     - 📊 Прогресс
     - 📞 Связаться с тренером
   - Для тренера (по ADMIN_ID):
     - 👥 Мои клиенты
     - ➕ Добавить программу

2. **Обработка кнопок и сценарии:**
   - **Моя программа:** Показывает текущую неделю тренировок с упражнениями, подходами, повторениями.
   - **Отметить выполнение:** Клиент отмечает выполненные упражнения.
   - **Прогресс:** Графики или статистика выполненных тренировок.
   - **Добавить программу (тренер):** Тренер выбирает клиента, загружает программу (текст + фото упражнений).

3. **Администрирование:**
   - Команда `/add_program [user_id] [program_json]` — назначить программу клиенту.
   - Команда `/list_clients` — показать всех клиентов с прогрессом.
   - Команда `/broadcast [текст]` — рассылка всем клиентам.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv, aiosqlite, aiofiles, asyncio.
   - **Структура проекта:**
```
/bot
├── main.py
├── config.py
├── handlers/
│   ├── client.py
│   └── trainer.py
├── keyboards/
│   └── main_menu.py
├── programs/ # папка для программ
├── database.py
├── .env
└── requirements.txt
```
   - **Конфигурация (.env):**
```
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
TRAINER_NAME=Иван Петров
TRAINER_PHONE=+79001234567
```
   - **Хранение данных:** SQLite с таблицами:
     - `programs` (id, user_id, trainer_id, program_json, week, created_at)
     - `workouts` (id, program_id, day, exercises_json, completed)
     - `users` (user_id, username, first_name, role, last_seen)

5. **Обработка ошибок и безопасность:**
   - Доступ к программам только у назначенных клиентов.
   - Сохранение истории выполнения.

6. **Деплой:**
   - Код готов к запуску через Docker или systemd.
   - Включить Dockerfile, docker-compose.yml, README.md.
