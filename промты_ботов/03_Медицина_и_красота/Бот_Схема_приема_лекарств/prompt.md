# Промт для создания Telegram-бота «Схема приема лекарств»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Схема приема лекарств».

**Контекст проекта:**
Этот бот продаётся врачам, клиникам и фармацевтам. Цель — помочь пациентам соблюдать график приёма лекарств: врач вносит схему (препарат, дозировка, время приёма), пациент получает ежедневные напоминания. Особенно полезно для хронических больных, пожилых людей и послеоперационных пациентов.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение.
   - Под сообщением — клавиатура (InlineKeyboardMarkup):
     - 💊 Мои лекарства
     - ⏰ Расписание на сегодня
     - ✅ Отметить приём
     - 📞 Связаться с врачом
     - ❓ Помощь

2. **Обработка кнопок и сценарии:**
   - **Мои лекарства:** Показывает список препаратов с дозировками и статусом («Активен», «Завершён»).
   - **Расписание на сегодня:** Выводит все приёмы лекарств на текущий день с временем и статусом («Принято», «Ожидается», «Пропущено»).
   - **Отметить приём:** Пациент отмечает, что принял лекарство. Бот спрашивает, какой препарат принят, и фиксирует время.
   - **Связаться с врачом:** Кнопка-ссылка на телефон врача.
   - **Помощь:** Инструкция.

3. **Администрирование (врач):**
   - Команда `/add_medication [user_id] [название] [дозировка] [время] [дни]` — добавить препарат пациенту. Время в формате HH:MM, дни через запятую (пн,вт,ср или daily).
   - Команда `/list_patients` — показать всех пациентов с активными схемами.
   - Команда `/stop_medication [user_id] [medication_id]` — завершить приём препарата.
   - Команда `/broadcast [текст]` — рассылка всем пациентам.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv, aiosqlite, asyncio, apscheduler (для напоминаний).
   - **Структура проекта:**
```
/bot
├── main.py
├── config.py
├── handlers/
│   ├── patient.py
│   └── doctor.py
├── keyboards/
│   └── main_menu.py
├── scheduler.py # планировщик напоминаний
├── database.py
├── .env
└── requirements.txt
```
   - **Конфигурация (.env):**
```
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
DOCTOR_NAME=Иван Петров
DOCTOR_PHONE=+79001234567
REMINDER_MINUTES_BEFORE=15
TIMEZONE=Europe/Moscow
```
   - **Хранение данных:** SQLite с таблицами:
     - `medications` (id, user_id, name, dosage, schedule_time, days_of_week, is_active, created_at)
     - `intakes` (id, medication_id, scheduled_time, taken_at, status)
     - `users` (user_id, username, first_name, last_seen)

5. **Обработка ошибок и безопасность:**
   - Напоминания отправляются точно по расписанию с учётом часового пояса.
   - Если пациент не отметил приём в течение 2 часов — повторное уведомление.
   - Логирование всех пропущенных приёмов для отчёта врачу.

6. **Деплой:**
   - Код готов к запуску через Docker или systemd.
   - Включить Dockerfile, docker-compose.yml, README.md.
