Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного Telegram-бота «Скидка на повторный прием».

**Контекст проекта:**
Бот для клиник и частных врачей. Автоматически высылает промокод на скидку пациентам через заданный период после первого визита (например, 3 месяца). Стимулирует возврат клиентов, повышает лояльность. Монетизация: продажа клиникам как инструмент маркетинга.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При `/start` бот приветствует пациента и предлагает подписаться на акции.
   - Меню пациента:
     - 🎁 Мои промокоды
     - 📅 История визитов
     - ❓ Условия акций
   - Меню админа:
     - 👥 Пациенты
     - 🏷️ Создать акцию
     - 📊 Статистика использования

2. **Обработка кнопок:**
   - **Мои промокоды (пациент):** Показывает активные промокоды с сроком действия.
   - **История визитов:** Список дат посещений клиники.
   - **Создать акцию (админ):**
     - Админ задает параметры: через сколько дней выслать, размер скидки, текст промокода.
     - Бот автоматически отслеживает пациентов и рассылает промокоды по расписанию.
   - **Статистика:** Сколько промокодов выдано, сколько использовано.

3. **Администрирование:**
   - Команда `/add_visit [user_id] [date]` — добавить запись о визите.
   - Команда `/list_promos` — список всех активных акций.
   - Настройка периода отправки (по умолчанию 90 дней).

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, sqlite3, python-dotenv, apscheduler.
   - **Структура проекта:**
/bot
├── main.py
├── config.py
├── handlers/
│ ├── patient.py
│ └── admin.py
├── keyboards/
│ └── menus.py
├── .env
└── requirements.txt
   - **Конфигурация (.env):**
BOT_TOKEN=...
ADMIN_IDS=123456789
DEFAULT_DISCOUNT_PERCENT=10
DEFAULT_DELAY_DAYS=90
CLINIC_NAME="Клиника Здоровья"

5. **Хранение данных:**
   - SQLite таблицы:
     - `users` (user_id, first_name, username, subscribed)
     - `visits` (id, user_id, visit_date)
     - `promos` (id, user_id, promo_code, discount_percent, issue_date, expiry_date, used)
     - `campaigns` (id, delay_days, discount_percent, active)

6. **Обработка ошибок и безопасность:**
   - Промокоды должны быть уникальными.
   - Проверка срока действия промокода при использовании.
   - Логирование всех выдач и использований.

7. **Деплой:**
   - Dockerfile, docker-compose.yml.
   - Инструкция по настройке планировщика задач.

**Формат ответа:**
Предоставь полный код всех файлов проекта. В конце дай инструкцию по развёртыванию.
