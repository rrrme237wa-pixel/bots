# Промт для создания Telegram-бота «Врач задерживается»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Врач задерживается».

**Контекст проекта:**
Этот бот продаётся частным клиникам и медицинским центрам. Цель — оперативно уведомлять пациентов о задержках приёма, чтобы снизить напряжённость в очереди и улучшить сервис. Администратор одним нажатием отправляет уведомления всем записанным на ближайший час пациентам с указанием времени задержки.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение с названием клиники.
   - Под сообщением — клавиатура (InlineKeyboardMarkup):
     - 📅 Моя запись
     - 🔔 Статус приёма
     - 📞 Контакты
     - ❓ Помощь

2. **Обработка кнопок и сценарии:**
   - **Моя запись:** Если пациент записан, бот показывает дату, время, врача и статус («Время приёма актуально» или «Врач задерживается на X минут»).
   - **Статус приёма:** Показывает текущий статус приёма пациента.
   - **Контакты:** Телефон регистратуры, адрес, кнопка «Позвонить».
   - **Помощь:** Инструкция.

3. **Администрирование:**
   - Команда `/add_appointment [user_id] [doctor] [datetime]` — добавить запись пациента.
   - Команда `/delay [minutes] [message]` — отправить уведомление всем записанным на ближайшие 2 часа о задержке на указанное количество минут. Опционально — кастомное сообщение.
   - Команда `/cancel_delay` — отменить уведомление о задержке (если врач успел).
   - Команда `/list_appointments` — показать все записи на сегодня.
   - Команда `/broadcast [текст]` — общая рассылка.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv, aiosqlite, asyncio, datetime.
   - **Структура проекта:**
```
/bot
├── main.py
├── config.py
├── handlers/
│   ├── patient.py
│   └── admin.py
├── keyboards/
│   └── main_menu.py
├── database.py
├── .env
└── requirements.txt
```
   - **Конфигурация (.env):**
```
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
CLINIC_NAME=«МедЦентр»
CLINIC_PHONE=+79001234567
CLINIC_ADDRESS=г. Москва, ул. Ленина, 1
NOTIFICATION_WINDOW_HOURS=2
```
   - **Хранение данных:** SQLite с таблицами:
     - `appointments` (id, user_id, username, doctor, appointment_datetime, status, delay_minutes, created_at)
     - `users` (user_id, username, first_name, last_seen)
     - `notifications` (id, user_id, message, sent_at, type)

5. **Обработка ошибок и безопасность:**
   - Рассылка с задержкой 0.05 сек между сообщениями.
   - Пропуск пользователей, заблокировавших бота.
   - Логирование всех отправленных уведомлений.

6. **Деплой:**
   - Код готов к запуску через Docker или systemd.
   - Включить Dockerfile, docker-compose.yml, README.md.
