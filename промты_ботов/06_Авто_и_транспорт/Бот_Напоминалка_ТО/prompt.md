Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Напоминалка ТО».

**Контекст проекта:**
Бот для автосервисов. Автоматически напоминает клиентам о необходимости планового ТО berdasarkan пробегу или времени. Увеличивает возвращаемость клиентов на 40%.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение с названием сервиса и текущими акциями на ТО.
   - Под сообщением — клавиатура с кнопками (ReplyKeyboardMarkup):
     - 📅 Записаться на ТО
     - 📊 Моя история
     - 💰 Прайс ТО
     - 📞 Контакты

2. **Обработка кнопок и команд:**
**Запись:** Выбор типа ТО (масло, фильтры, колодки), даты и времени.
**Напоминания:** Клиент вносит данные авто (марка, пробег, дата последнего ТО). Бот рассчитывает следующее ТО.
**История:** Все прошлые визиты с чеками.

3. **Администрирование:**
/add_client - добавить клиента, /reminders_today - кому напомнить, /stats - статистика

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv для конфигурации, aiofiles для работы с файлами, asyncio, aioschedule для планирования
   - **Структура проекта:**
```
/bot
├── main.py              # точка входа, инициализация бота и диспетчера
├── config.py            # загрузка настроек из .env и JSON файлов
├── handlers/
│   ├── user.py          # обработчики пользовательских команд и кнопок
│   └── admin.py         # обработчики админских команд (/admin, /broadcast)
├── keyboards/
│   └── main_menu.py     # функции создания Inline/Reply клавиатур
├── data/
│   ├── to_types.json, prices.json
│   └── service_photos/
├── .env                 # токен бота, ID админа и другие настройки
└── requirements.txt
```
   - **Конфигурация (.env):**
```
BOT_TOKEN=123456:ABCdefGHIjklMNOpqrsTUVwxyz
ADMIN_ID=123456789
SERVICE_NAME=АвтоМастер
REMINDER_KM=10000
REMINDER_MONTHS=6
```

5. **Хранение данных:**
SQLite: clients(user_id,car_model,last_km,last_to_date,next_reminder), visits(client_id,date,type,cost)

6. **Обработка ошибок и безопасность:**
   - Все внешние запросы должны иметь таймауты (не более 10 сек) и обработку исключений.
   - При ошибке чтения файлов или БД бот должен отправлять пользователю вежливое сообщение «Сервис временно недоступен, попробуйте позже», а админу — детали ошибки в ЛС.
Рассылка не чаще 1 раза в неделю. Возможность отписаться от напоминаний.

7. **Деплой:**
   - Код должен быть готов к запуску на сервере Ubuntu через systemd или Docker.
   - Напиши пример Dockerfile и docker-compose.yml для простоты развёртывания.
   - Включи инструкцию по первому запуску и настройке в файл `README.md`.
