# Промт для создания Telegram-бота «Сегодня в меню»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Сегодня в меню».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для кафе, столовых, ресторанов и кофейнь. Цель — ежедневно рассылать подписчикам актуальное меню бизнес-ланчей или специальных предложений с фотографиями. Администратор просто выбирает картинки из галереи бота, не тратя время на вёрстку и публикацию.

**Монетизация:** Продажа готового решения заведениям за 4000–6000 рублей, либо ежемесячная подписка 700 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение от имени заведения.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - 🍽️ Сегодня в меню
  - 📅 Меню на неделю
  - 🔔 Подписаться на рассылку
  - 📞 Контакты

### 2. Обработка кнопок

#### Сегодня в меню
- Бот показывает текущее меню дня с категориями:
  - Салаты
  - Супы
  - Горячие блюда
  - Напитки
  - Десерты
- Каждое блюдо сопровождается фото, описанием, ценой и объёмом порции.
- Кнопка «Заказать» (переход на доставку или телефон).

#### Меню на неделю
- Выбор дня недели (Пн–Вс).
- Показ меню на выбранный день.

#### Подписаться на рассылку
- Клиент подписывается на ежедневную рассылку меню.
- Выбор времени получения (11:00, 12:00 и т.д.).
- Кнопка «Отписаться».

### 3. Администрирование

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — статистика:
  - Количество подписчиков
  - Открытия меню за день
  - Популярные блюда

- `/set_menu [день]` — установить меню на день:
  1. Загрузка фото блюд
  2. Ввод названий, описаний, цен
  3. Публикация

- `/send_today` — отправить рассылку с меню на сегодня.

- `/schedule_auto` — настроить автоматическую рассылку.

- `/broadcast [текст]` — общая рассылка.

### 4. Технические детали

**Стек:**
- Python 3.10+
- aiogram 3.x
- python-dotenv для конфигурации
- aiosqlite для хранения данных
- apscheduler для авто-рассылки
- asyncio

**Структура проекта:**
```
/bot
├── main.py                    # точка входа
├── config.py                  # загрузка настроек из .env
├── handlers/
│   ├── user.py                # обработчики пользовательских команд
│   ├── admin.py               # обработчики админских команд
│   └── menu.py                # логика меню
├── keyboards/
│   └── main_menu.py           # главное меню
├── utils/
│   ├── database.py            # работа с SQLite
│   └── scheduler.py           # планировщик рассылок
├── data/
│   ├── menu_photos/           # фото блюд
│   └── database.db            # SQLite база данных
├── .env                       # токен бота, ID админа и т.д.
├── requirements.txt
└── README.md                  # инструкция по запуску
```

**Конфигурация (.env):**
```env
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
RESTAURANT_NAME=Столовая «Домашняя»
DELIVERY_PHONE=+79001234567
AUTO_SEND_HOUR=11
TIMEZONE=Europe/Moscow
```

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `subscribed` BOOLEAN DEFAULT 0
- `preferred_time` TIME
- `first_seen` TIMESTAMP

Таблица `menu_items`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `day_of_week` INTEGER (0–6)
- `category` TEXT
- `name` TEXT
- `description` TEXT
- `price` INTEGER
- `photo_path` TEXT
- `weight` TEXT
- `is_available` BOOLEAN DEFAULT 1

При каждом `/start` обновлять запись в `users`.

### 6. Обработка ошибок и безопасность

- Таймауты на запросы (10 сек).
- Авто-рассылка с задержкой 0.05 сек.
- Обработка `TelegramForbiddenError`.

### 7. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
```

**Инструкция по развёртыванию:**
1. Скопировать файлы на сервер
2. Создать `.env`
3. `docker-compose up -d`
