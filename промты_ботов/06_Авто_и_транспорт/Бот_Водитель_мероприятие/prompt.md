Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Водитель на мероприятие».

**Контекст проекта:**
Сервис заказа водителей с личным авто для мероприятий (свадьбы, корпоративы, трансферы). Клиенты выбирают класс авто, дату и время. Водители получают заявки.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение с названием сервиса, преимуществами (пунктуальность, опытные водители) и кнопками выбора услуг.
   - Под сообщением — клавиатура с кнопками (InlineKeyboardMarkup с категориями услуг):
     - 🚗 Легковое авто
     - 🚐 Минивэн
     - 🚌 Автобус
     - 📅 Мои заказы
     - 📞 Связаться

2. **Обработка кнопок и команд:**
**Выбор авто:** Каталог с фото, описанием, вместимостью, ценой за час.
**Заказ:** FSM опросник: дата, время, маршрут, длительность. Расчёт итоговой стоимости.
**Мои заказы:** История заказов со статусами (новый, подтверждён, выполнен).

3. **Администрирование:**
/admin - статистика заказов, /drivers - список водителей, /broadcast - рассылка

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv для конфигурации, aiofiles для работы с файлами, asyncio, aiohttp для внешних запросов, pillow для обработки фото
   - **Структура проекта:**
```
/bot
├── main.py              # точка входа, инициализация бота и диспетчера
├── config.py            # загрузка настроек из .env и JSON файлов
├── handlers/
│   ├── user.py          # обработчики пользовательских команд и кнопок
│   └── admin.py         # обработчики админских команд (/admin, /broadcast)
├── keyboards/
│   └── main_menu.py     # функции создания Inline/Reply клавиатур
├── data/
│   ├── vehicles.json (автопарк с характеристиками)
│   └── vehicles_photos/
├── .env                 # токен бота, ID админа и другие настройки
└── requirements.txt
```
   - **Конфигурация (.env):**
```
BOT_TOKEN=123456:ABCdefGHIjklMNOpqrsTUVwxyz
ADMIN_ID=123456789
SERVICE_NAME=AutoDrive
MIN_BOOKING_HOURS=3
BASE_RATE_PER_HOUR=1500
```

5. **Хранение данных:**
SQLite: users(user_id,role), orders(id,user_id,vehicle_id,date,hours,total,status), vehicles(id,name,capacity,rate_per_hour)

6. **Обработка ошибок и безопасность:**
   - Все внешние запросы должны иметь таймауты (не более 10 сек) и обработку исключений.
   - При ошибке чтения файлов или БД бот должен отправлять пользователю вежливое сообщение «Сервис временно недоступен, попробуйте позже», а админу — детали ошибки в ЛС.
Проверка доступности авто перед подтверждением. Блокировка пользователей после 3 отмен без причины.

7. **Деплой:**
   - Код должен быть готов к запуску на сервере Ubuntu через systemd или Docker.
   - Напиши пример Dockerfile и docker-compose.yml для простоты развёртывания.
   - Включи инструкцию по первому запуску и настройке в файл `README.md`.
