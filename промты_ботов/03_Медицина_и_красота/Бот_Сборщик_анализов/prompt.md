# Промт для создания Telegram-бота «Сборщик анализов»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Сборщик анализов».

**Контекст проекта:**
Этот бот продаётся частным клиникам, лабораториям и врачам. Цель — упростить процесс получения результатов анализов от пациентов: пациент фотографирует бланк и отправляет в бота, бот пересылает врачу в организованном виде с метаданными (дата, имя пациента, тип анализа). Это экономит время на сортировку и снижает риск потери результатов.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение с инструкцией.
   - Под сообщением — клавиатура (ReplyKeyboardMarkup):
     - 📤 Отправить анализ
     - 📋 Мои анализы
     - 📞 Связаться с врачом
     - ❓ Помощь

2. **Обработка кнопок и сценарии:**
   - **Отправить анализ:** Бот запрашивает тип анализа (выбор из списка или ввод текста), затем просит загрузить фото/файл. После загрузки бот сохраняет файл в папку `analyses/[user_id]/[timestamp]_analysis.jpg` и отправляет подтверждение. Админ (врач) получает уведомление с файлом и данными пациента.
   - **Мои анализы:** Показывает историю загруженных анализов пользователя с датами и статусами («На проверке», «Готов», «Требуется пересдача»). Статусы меняет админ.
   - **Связаться с врачом:** Кнопка-ссылка на телефон или личные сообщения врача.
   - **Помощь:** Инструкция по использованию.

3. **Администрирование:**
   - Команда `/list_analyses` — показать все новые анализы за сегодня.
   - Команда `/set_status [user_id] [status]` — установить статус анализа пациента.
   - Команда `/broadcast [текст]` — рассылка всем пациентам.
   - Команда `/export` — выгрузить все анализы за период в ZIP-архиве.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv, aiosqlite, aiofiles, asyncio.
   - **Структура проекта:**
```
/bot
├── main.py
├── config.py
├── handlers/
│   ├── user.py
│   └── admin.py
├── keyboards/
│   └── main_menu.py
├── data/
│   └── analysis_types.json
├── analyses/ # папка для файлов
├── database.py
├── .env
└── requirements.txt
```
   - **Конфигурация (.env):**
```
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
DOCTOR_NAME=Иван Петров
DOCTOR_PHONE=+79001234567

```
   - **Хранение данных:** SQLite с таблицами:
     - `analyses` (id, user_id, username, analysis_type, file_path, status, created_at, updated_at)
     - `users` (user_id, username, first_name, last_seen)

5. **Обработка ошибок и безопасность:**
   - Ограничение на размер файла (проверка перед загрузкой).
   - Валидация типов файлов (только изображения и PDF).
   - При ошибке — вежливое сообщение пользователю, лог админу.

6. **Деплой:**
   - Код готов к запуску через Docker или systemd.
   - Включить Dockerfile, docker-compose.yml, README.md.
