# Промт для создания Telegram-бота «Челлендж»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Челлендж».

**Контекст проекта:**
Этот бот продаётся фитнес-тренерам, ЗОЖ-блогерам и марафонам похудения. Цель — проводить месячные челленджи: участники ежедневно получают задания, отправляют фотоотчёты, бот ведёт статистику и определяет победителей. Монетизация через платный вход.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение с описанием текущего челленджа.
   - Под сообщением — клавиатура (InlineKeyboardMarkup):
     - 📅 Задание на сегодня
     - 📸 Отправить отчёт
     - 🏆 Таблица лидеров
     - 💳 Участвовать в челлендже

2. **Обработка кнопок и сценарии:**
   - **Задание на сегодня:** Бот показывает задание дня (текст + фото/видео примера).
   - **Отправить отчёт:** Участник загружает фото/видео выполнения. Бот сохраняет и подтверждает.
   - **Таблица лидеров:** Рейтинг участников по количеству выполненных заданий.
   - **Участвовать:** Оплата и регистрация в челлендже.

3. **Администрирование:**
   - Команда `/create_challenge [name] [start_date] [end_date] [price]` — создать челлендж.
   - Команда `/add_task [day] [description] [media]` — добавить задание.
   - Команда `/list_participants` — показать всех участников.
   - Команда `/verify_report [user_id] [task_id]` — подтвердить отчёт.
   - Команда `/broadcast [текст]` — рассылка участникам.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv, aiosqlite, aiofiles, asyncio, apscheduler.
   - **Структура проекта:**
```
/bot
├── main.py
├── config.py
├── handlers/
│   ├── participant.py
│   └── admin.py
├── keyboards/
│   └── main_menu.py
├── reports/ # папка для отчётов
├── database.py
├── scheduler.py
├── .env
└── requirements.txt
```
   - **Конфигурация (.env):**
```
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
CHALLENGE_NAME=30 дней спорта
PAYMENT_URL=https://payment.ru/...
TIMEZONE=Europe/Moscow
```
   - **Хранение данных:** SQLite с таблицами:
     - `challenges` (id, name, start_date, end_date, price, is_active)
     - `participants` (id, challenge_id, user_id, joined_at, total_points)
     - `tasks` (id, challenge_id, day, description, media_path)
     - `reports` (id, task_id, user_id, media_path, status, submitted_at)

5. **Обработка ошибок и безопасность:**
   - Автоматическая публикация заданий по расписанию.
   - Модерация отчётов админом.
   - Подсчёт баллов и обновление рейтинга.

6. **Деплой:**
   - Код готов к запуску через Docker или systemd.
   - Включить Dockerfile, docker-compose.yml, README.md.
