# Промт для создания Telegram-бота «Калькулятор банкета»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Калькулятор банкета».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для ресторанов, кафе, банкетных залов и кейтеринговых компаний. Цель — позволить клиентам самостоятельно рассчитать примерную стоимость банкета, выбирая пакеты услуг, количество гостей и дополнительные опции. Это экономит время менеджеров на первичных консультациях.

**Монетизация:** Продажа готового решения заведениям за 5000–8000 рублей, либо ежемесячная подписка 900 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение от имени заведения.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - 🧮 Рассчитать стоимость
  - 📋 Пакеты услуг
  - 📅 Забронировать дату
  - 📞 Контакты

### 2. Обработка кнопок

#### Рассчитать стоимость
- Бот запускает FSM для пошагового расчёта:
  1. **Тип мероприятия:** Выбор из списка (свадьба, корпоратив, день рождения, конференция).
  2. **Количество гостей:** Ввод числа (10–500).
  3. **Дата мероприятия:** Выбор из календаря.
  4. **Пакет услуг:** Выбор из вариантов (Эконом, Стандарт, Премиум, VIP).
  5. **Дополнительные опции:** Мультивыбор (живая музыка, декор, фото/видео, торт, трансфер).
  6. **Спецпожелания:** Текстовое поле.
- После заполнения бот показывает детальный расчёт:
  - Стоимость пакета на человека × количество гостей
  - Стоимость дополнительных опций
  - Итоговая сумма
  - Срок действия расчёта (7 дней)
- Кнопка «Заказать банкет» — сохраняет заявку и отправляет менеджеру.

#### Пакеты услуг
- Описание каждого пакета с составом:
  - Эконом: фуршет, напитки, базовое оформление
  - Стандарт: 3 блюда, напитки, музыка
  - Премиум: 5 блюд, премиум-напитки, шоу-программа
  - VIP: индивидуальное меню, персональный сервис

#### Забронировать дату
- Проверка доступности дат.
- Заявка на бронь (менеджер связывается для подтверждения).

### 3. Администрирование

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — статистика:
  - Количество расчётов за период
  - Средний чек
  - Конверсия в заявки

- `/requests` — список заявок на банкет.

- `/update_package [id] [price]` — изменить цену пакета.

- `/export` — выгрузка в CSV.

### 4. Технические детали

**Стек:**
- Python 3.10+
- aiogram 3.x
- python-dotenv для конфигурации
- aiosqlite для хранения данных
- asyncio

**Структура проекта:**
```
/bot
├── main.py                    # точка входа
├── config.py                  # загрузка настроек из .env
├── handlers/
│   ├── user.py                # обработчики пользовательских команд
│   ├── admin.py               # обработчики админских команд
│   └── calculator.py          # логика калькулятора (FSM)
├── keyboards/
│   └── main_menu.py           # главное меню
├── utils/
│   ├── database.py            # работа с SQLite
│   └── calculator.py          # математическая логика
├── data/
│   └── database.db            # SQLite база данных
├── .env                       # токен бота, ID админа и т.д.
├── requirements.txt

```

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `first_seen` TIMESTAMP

Таблица `packages`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `name` TEXT
- `price_per_person` INTEGER
- `description` TEXT

Таблица `options`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `name` TEXT
- `price` INTEGER

Таблица `calculations`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER
- `event_type` TEXT
- `guests_count` INTEGER
- `package_id` INTEGER
- `options` TEXT (JSON)
- `total_price` INTEGER
- `created_at` TIMESTAMP

При каждом `/start` обновлять запись в `users`.

### 6. Обработка ошибок и безопасность

- Таймауты на запросы (10 сек).
- Валидация количества гостей (min/max).
- Защита от отрицательных значений.

### 7. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
```

**Инструкция по развёртыванию:**
1. Скопировать файлы на сервер
2. Создать `.env`
3. `docker-compose up -d`
