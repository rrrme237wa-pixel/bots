# Промт для создания Telegram-бота «Предзапись на новую коллекцию»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Предзапись на новую коллекцию».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для салонов красоты, барбершопов, студий маникюра и других заведений, которые периодически запускают новые услуги или коллекции. Цель — собрать лист ожидания из заинтересованных клиентов до официального открытия записи, чтобы обеспечить заполненность в первые дни запуска.

**Монетизация:** Продажа готового решения салонам за 4000–7000 рублей, либо ежемесячная подписка 600 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение с информацией о текущей акции/новой коллекции.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - 📋 Встать в лист ожидания
  - 🔔 Узнать о старте первым
  - 👤 Мой статус
  - ℹ️ О коллекции

### 2. Обработка кнопок

#### Встать в лист ожидания
- Бот запрашивает имя и контактный телефон (если не получен автоматически).
- Клиент выбирает предпочитаемую услугу из новой коллекции.
- Выбор желаемой даты/времени (приоритетные слоты для участников листа ожидания).
- После подтверждения клиент добавляется в базу листа ожидания.
- Бот выдаёт уникальный номер в списке (например: «Вы №15 в листе ожидания»).

#### Узнать о старте первым
- Клиент подписывается на уведомление об открытии записи.
- При открытии записи все подписчики получают пуш-уведомление одновременно.

#### Мой статус
- Отображение текущей позиции в листе ожидания.
- Информация о дате открытия записи.
- Кнопка «Выйти из листа ожидания».

#### О коллекции
- Описание новой коллекции/услуги.
- Фотографии работ (медиагруппа).
- Цены (или диапазон цен).
- Дата официального запуска.

### 3. Администрирование (для владельца салона)

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — статистика:
  - Количество участников листа ожидания
  - Количество подписчиков на уведомление
  - Популярные услуги

- `/waiting_list` — просмотр полного списка участников с контактами.

- `/export_waiting` — выгрузка листа ожидания в CSV/Excel.

- `/open_booking` — открытие записи (рассылка всем подписчикам).

- `/broadcast [текст]` — рассылка участникам.

- `/add_manual [phone] [service]` — добавить участника вручную (для звонков).

### 4. Технические детали

**Стек:**
- Python 3.10+
- aiogram 3.x
- python-dotenv для конфигурации
- aiosqlite для хранения данных
- asyncio

**Структура проекта:**
```
/bot
├── main.py                    # точка входа
├── config.py                  # загрузка настроек из .env
├── handlers/
│   ├── user.py                # обработчики пользовательских команд
│   ├── admin.py               # обработчики админских команд
│   └── waiting_list.py        # логика листа ожидания
├── keyboards/
│   ├── main_menu.py           # главное меню
│   └── collection_kb.py       # клавиатуры коллекции
├── utils/
│   ├── database.py            # работа с SQLite
│   └── notifications.py       # уведомления
├── data/
│   ├── collection_photos/     # фото новой коллекции
│   └── database.db            # SQLite база данных
├── .env                       # токен бота, ID админа и т.д.
├── requirements.txt
└── README.md                  # инструкция по запуску
```

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `phone` TEXT
- `first_seen` TIMESTAMP

Таблица `waiting_list`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER
- `service_type` TEXT
- `preferred_date` DATETIME
- `position` INTEGER
- `joined_at` TIMESTAMP
- `is_active` BOOLEAN DEFAULT 1

Таблица `notifications`:
- `user_id` INTEGER PRIMARY KEY
- `subscribed_at` TIMESTAMP
- `notified_at` TIMESTAMP

При каждом `/start` обновлять запись в `users`.

### 6. Обработка ошибок и безопасность

- Таймауты на запросы (10 сек).
- Рассылка при открытии записи с задержкой 0.05 сек.
- Обработка `TelegramForbiddenError`.
- Защита от дублирования записей в листе ожидания.

### 7. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
```

**Инструкция по развёртыванию:**
1. Скопировать файлы на сервер
2. Создать `.env`
3. `docker-compose up -d`
