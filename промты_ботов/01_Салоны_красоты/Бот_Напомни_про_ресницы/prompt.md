# Промт для создания Telegram-бота «Напомни про ресницы»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Напомни про ресницы».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для мастеров по наращиванию ресниц и ламинированию. Цель — автоматизировать напоминания клиентам о коррекции, чтобы повысить возвращаемость клиентов и сэкономить время мастера на ручных SMS/звонках.

**Монетизация:** Продажа готового решения мастерам за 4000–6000 рублей, либо ежемесячная подписка 700 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение от имени салона/мастера.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - 📅 Записаться на коррекцию
  - 🔔 Настроить напоминания
  - 👤 Мой профиль
  - 📞 Контакты

### 2. Обработка кнопок

#### Записаться на коррекцию
- Бот запрашивает у клиента желаемую дату и время (через FSM).
- После подтверждения запись сохраняется в базе данных.
- Мастер получает уведомление о новой записи.
- Клиенту автоматически устанавливается напоминание за 2 дня до даты коррекции.

#### Настроить напоминания
- Клиент может включить/выключить автоматические напоминания.
- Выбор периода напоминания: за 1 день, за 2 дня, за 3 дня до коррекции.
- Выбор времени отправки напоминания (утро/день/вечер).

#### Мой профиль
- Отображение информации о клиенте:
  - Дата последней процедуры
  - Дата следующей рекомендуемой коррекции
  - История посещений (последние 5 записей)
  - Статус напоминаний (вкл/выкл)

#### Контакты
- Телефон мастера/салона
- Адрес с кнопкой «Проложить маршрут» (Яндекс.Карты/Google Maps)
- Ссылка на Instagram/WhatsApp

### 3. Администрирование (для мастера)

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — показывает статистику:
  - Количество активных клиентов
  - Количество записей на сегодня/неделю
  - Список клиентов, которым нужно отправить напоминания сегодня

- `/add_client [user_id] [дата]` — вручную добавить клиента и дату коррекции (для записи через звонок).

- `/broadcast [текст]` — рассылка всем клиентам (с задержкой 0.05 сек).

- `/export_clients` — выгрузка списка клиентов с датами последних визитов в CSV.

- `/remind_today` — принудительная отправка напоминаний клиентам, у которых коррекция через N дней.

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `last_name` TEXT
- `phone` TEXT
- `first_seen` TIMESTAMP
- `last_seen` TIMESTAMP
- `reminders_enabled` BOOLEAN DEFAULT 1
- `reminder_days_before` INTEGER DEFAULT 2

Таблица `appointments`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER
- `appointment_date` DATE
- `appointment_time` TIME
- `status` TEXT (scheduled/completed/cancelled)
- `created_at` TIMESTAMP
- `reminder_sent` BOOLEAN DEFAULT 0

Таблица `visit_history`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER
- `visit_date` DATE
- `procedure_type` TEXT
- `notes` TEXT

При каждом `/start` добавлять или обновлять запись в `users`.

### 6. Логика напоминаний

**Планировщик (APScheduler):**
- Ежедневно в заданное время (например, 10:00) проверять базу данных.
- Находить клиентов, у которых `appointment_date` через N дней (где N = `reminder_days_before`).
- Отправлять персонализированное сообщение:
  ```
  Добрый день, {имя}! 🌸
  
  Напоминаем, что через {N} дня вам запланирована коррекция ресниц.
  
  📅 Дата: {дата}
  ⏰ Время: {время}
  
  Для подтверждения нажмите ✅, для переноса — 📅.
  ```
- При подтверждении обновлять статус записи.
- При переносе — предлагать свободные слоты.

### 7. Обработка ошибок и безопасность

- Все внешние запросы должны иметь таймауты (10 сек) и обработку исключений.
- При ошибке отправки напоминания логировать ошибку и продолжать обработку следующих клиентов.
- Рассылка должна иметь искусственную задержку 0.05 сек между сообщениями.
- Обработка `TelegramForbiddenError` — удалять пользователя из активной рассылки.
- Валидация дат (нельзя записаться в прошлое).

### 8. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
```

**systemd сервис:**
```ini
[Unit]
Description=Lash Reminder Bot
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/lash-bot
ExecStart=/usr/bin/python3 main.py
Restart=always

[Install]
WantedBy=multi-user.target
```

**Инструкция по развёртыванию на VPS:**

1. Скопировать файлы проекта на сервер
2. Создать файл `.env` с необходимыми переменными
3. Запустить через Docker: `docker-compose up -d`
4. Или через systemd:
   ```bash
   sudo cp lash-bot.service /etc/systemd/system/
   sudo systemctl daemon-reload
   sudo systemctl enable lash-bot
   sudo systemctl start lash-bot
   ```

5. Проверить логи: `journalctl -u lash-bot -f`
