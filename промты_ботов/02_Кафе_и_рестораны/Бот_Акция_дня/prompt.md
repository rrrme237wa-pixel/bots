# Промт для создания Telegram-бота «Акция дня»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Акция дня».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для кофеен, пекарен, фастфудов и других заведений быстрого питания. Цель — каждое утро выставлять скидку на определённую позицию меню, чтобы стимулировать продажи в низкие часы и привлекать подписчиков специальными предложениями.

**Монетизация:** Продажа готового решения заведениям за 3000–5000 рублей, либо ежемесячная подписка 500 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение от имени заведения.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - 🔥 Акция дня
  - 📅 Акции недели
  - 🔔 Подписаться на уведомления
  - 📞 Контакты

### 2. Обработка кнопок

#### Акция дня
- Бот показывает текущую акцию:
  - Фото товара
  - Название и описание
  - Старая цена (зачёркнута)
  - Новая цена со скидкой
  - Время действия акции (например, до 14:00)
  - QR-код для предъявления бариста
- Кнопка «Показать QR» — генерирует уникальный код для списания.

#### Акции недели
- Календарь акций на неделю (Пн–Вс).
- Для каждого дня — своя позиция со скидкой.

#### Подписаться на уведомления
- Клиент подписывается на ежедневные пуши об акциях.
- Выбор времени получения (8:00, 9:00, 10:00).

### 3. Администрирование

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — статистика:
  - Количество подписчиков
  - Использование акций за день
  - Популярные товары

- `/set_promo [товар] [скидка] [время]` — установить акцию на сегодня.

- `/send_promo` — отправить уведомление подписчикам.

- `/schedule_week` — запланировать акции на неделю.

- `/export` — выгрузка в CSV.

### 4. Технические детали

**Стек:**
- Python 3.10+
- aiogram 3.x
- python-dotenv для конфигурации
- aiosqlite для хранения данных
- apscheduler для авто-рассылки
- qrcode для генерации QR
- asyncio

**Структура проекта:**
```
/bot
├── main.py                    # точка входа
├── config.py                  # загрузка настроек из .env
├── handlers/
│   ├── user.py                # обработчики пользовательских команд
│   ├── admin.py               # обработчики админских команд
│   └── promo.py               # логика акций
├── keyboards/
│   └── main_menu.py           # главное меню
├── utils/
│   ├── database.py            # работа с SQLite
│   ├── scheduler.py           # планировщик
│   └── qr_generator.py        # генерация QR
├── data/
│   └── database.db            # SQLite база данных
├── .env                       # токен бота, ID админа и т.д.
├── requirements.txt
└── README.md                  # инструкция по запуску
```

**Конфигурация (.env):**
```env
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
CAFE_NAME=Кофейня «Morning»
AUTO_SEND_HOUR=8
TIMEZONE=Europe/Moscow
```

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `subscribed` BOOLEAN DEFAULT 0
- `preferred_time` TIME
- `first_seen` TIMESTAMP

Таблица `promos`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `product_name` TEXT
- `product_photo` TEXT
- `old_price` INTEGER
- `discount_percent` INTEGER
- `new_price` INTEGER
- `valid_until` DATETIME
- `is_active` BOOLEAN DEFAULT 1
- `created_at` TIMESTAMP

Таблица `promo_usage`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER
- `promo_id` INTEGER
- `used_at` TIMESTAMP

При каждом `/start` обновлять запись в `users`.

### 6. Обработка ошибок и безопасность

- Таймауты на запросы (10 сек).
- Уникальность QR-кодов.
- Защита от повторного использования акции одним пользователем.
- Авто-рассылка с задержкой 0.05 сек.

### 7. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
```

**Инструкция по развёртыванию:**
1. Скопировать файлы на сервер
2. Создать `.env`
3. `docker-compose up -d`
