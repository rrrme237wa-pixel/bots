# Промт для создания Telegram-бота «Результаты УЗИ»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Результаты УЗИ».

**Контекст проекта:**
Этот бот продаётся частным клиникам и врачам УЗИ. Цель — безопасно и конфиденциально передавать пациентам результаты исследований (файлы DICOM, PDF, изображения) без использования email и сторонних облаков. Врач загружает файл в бота, бот генерирует одноразовую ссылку для пациента, которая действует 24 часа.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение.
   - Для пациента — клавиатура (InlineKeyboardMarkup):
     - 📄 Получить результаты
     - ℹ️ Как это работает
     - 📞 Контакты клиники
   - Для врача (по ADMIN_ID) — дополнительная кнопка:
     - 👨‍⚕️ Загрузить результат

2. **Обработка кнопок и сценарии:**
   - **Получить результаты:** Пациент вводит код из SMS или QR-кода, который выдала клиника. Бот проверяет код и выдаёт файл (если срок действия не истёк). После получения код сгорает.
   - **Загрузить результат (врач):** Врач вводит ID пациента или номер записи, загружает файл (PDF/DICOM/JPG), бот генерирует уникальный код доступа и сохраняет запись.
   - **Как это работает:** Инструкция для пациентов.
   - **Контакты клиники:** Телефон, адрес, режим работы.

3. **Администрирование:**
   - Команда `/upload [patient_id]` — начать процесс загрузки результата для пациента.
   - Команда `/list_results` — показать все загруженные результаты за период.
   - Команда `/revoke [code]` — аннулировать код доступа досрочно.
   - Команда `/stats` — статистика: всего выдано, получено, просрочено.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv, aiosqlite, aiofiles, asyncio, secrets (для генерации кодов).
   - **Структура проекта:**
```
/bot
├── main.py
├── config.py
├── handlers/
│   ├── patient.py
│   └── doctor.py
├── keyboards/
│   └── main_menu.py
├── results/ # папка для файлов результатов
├── database.py
├── .env
└── requirements.txt
```
   - **Конфигурация (.env):**
```
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
CLINIC_NAME=«УЗИ-Центр»
CLINIC_PHONE=+79001234567
CLINIC_ADDRESS=г. Москва, ул. Мира, 5
CODE_EXPIRY_HOURS=24
MAX_FILE_SIZE_MB=50
```
   - **Хранение данных:** SQLite с таблицами:
     - `results` (id, patient_id, doctor_id, file_path, access_code, is_used, expires_at, created_at)
     - `patients` (user_id, username, first_name, phone, last_seen)

5. **Обработка ошибок и безопасность:**
   - Коды доступа — уникальные строки 8 символов (letters+digits).
   - Файлы хранятся на сервере, доступ только через бота.
   - Автоматическая очистка старых файлов (cron или фоновая задача).
   - Логирование всех выдач файлов.

6. **Деплой:**
   - Код готов к запуску через Docker или systemd.
   - Включить Dockerfile, docker-compose.yml, README.md.
