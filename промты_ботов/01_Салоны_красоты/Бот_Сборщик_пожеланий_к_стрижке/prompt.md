# Промт для создания Telegram-бота «Сборщик пожеланий к стрижке»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Сборщик пожеланий к стрижке».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для парикмахеров, барбершопов и салонов красоты. Цель — позволить клиентам заранее отправить фото желаемой причёски и текстовые пожелания, чтобы мастер мог подготовиться к визиту и сократить время на обсуждение деталей на месте.

**Монетизация:** Продажа готового решения салонам за 5000–8000 рублей, либо ежемесячная подписка 800 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение от имени салона/мастера.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - ✂️ Создать карточку пожеланий
  - 📋 Мои предыдущие заявки
  - 📅 Записаться на стрижку
  - ℹ️ Как это работает

### 2. Обработка кнопок

#### Создать карточку пожеланий
- Бот запускает FSM (Finite State Machine) для пошагового сбора информации:
  1. **Тип услуги:** Выбор из списка (мужская стрижка, женская стрижка, окрашивание, укладка, борода).
  2. **Длина волос:** Короткие/средние/длинные.
  3. **Фото примера:** Клиент загружает 1–5 фотографий желаемой причёски.
  4. **Текстовые пожелания:** Клиент описывает детали (чёлка, градуировка, цвет и т.д.).
  5. **Комментарии:** Особые указания (чувствительная кожа, аллергия на краску и т.п.).
  6. **Дата визита:** Выбор даты и времени записи.
- После заполнения бот формирует итоговую карточку с предпросмотром.
- Клиент подтверждает отправку или редактирует данные.
- Мастер получает уведомление с полной карточкой пожеланий.

#### Мои предыдущие заявки
- Отображение истории заявок клиента.
- Для каждой заявки:
  - Дата создания
  - Тип услуги
  - Статус (новая/в работе/выполнена)
  - Кнопка «Повторить заявку» (копирует данные в новую).

#### Записаться на стрижку
- Быстрая запись без заполнения пожеланий.
- Выбор даты и времени из свободных слотов.
- Подтверждение записи.

#### Как это работает
- Информационное сообщение с инструкцией:
  ```
  1. Нажмите «Создать карточку пожеланий»
  2. Выберите тип услуги
  3. Загрузите фото примера (можно из галереи)
  4. Опишите ваши пожелания текстом
  5. Подтвердите отправку мастеру
  
  Мастер изучит вашу заявку до визита и будет готов!
  ```

### 3. Администрирование (для мастера)

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — показывает статистику:
  - Количество новых заявок сегодня
  - Количество активных заявок
  - Список предстоящих визитов

- `/view_requests` — просмотр всех активных заявок в формате:
  ```
  🔔 Новая заявка #123
  Клиент: @username
  Услуга: Женская стрижка
  Дата визита: 15.10.2024 14:00
  
  [Фото]
  Пожелания: Хочу каре до плеч с чёлкой
  ```

- `/update_status [id] [status]` — изменение статуса заявки (new/in_progress/completed).

- `/broadcast [текст]` — рассылка всем клиентам.

- `/export_requests` — выгрузка всех заявок в CSV.

### 4. Технические детали

**Стек:**
- Python 3.10+
- aiogram 3.x
- python-dotenv для конфигурации
- aiosqlite для хранения данных
- asyncio
- aiofiles для работы с файлами

**Структура проекта:**
```

│   ├── admin.py               # обработчики админских команд
│   └── wishlist.py            # логика сбора пожеланий (FSM)
```

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `last_name` TEXT
- `phone` TEXT
- `first_seen` TIMESTAMP
- `last_seen` TIMESTAMP

Таблица `requests`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER
- `service_type` TEXT
- `hair_length` TEXT
- `description` TEXT
- `comments` TEXT
- `visit_date` DATETIME
- `status` TEXT (new/in_progress/completed/cancelled)
- `created_at` TIMESTAMP

Таблица `request_photos`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `request_id` INTEGER
- `file_id` TEXT (Telegram file_id)
- `file_path` TEXT (локальный путь)
- `uploaded_at` TIMESTAMP

При каждом `/start` добавлять или обновлять запись в `users`.

### 6. Обработка медиафайлов

- Загруженные фотографии сохраняются в папку `data/uploads/{user_id}/`.
- Каждому файлу присваивается уникальное имя: `{timestamp}_{random}.jpg`.
- В базе данных хранится `file_id` Telegram для быстрого доступа.
- При удалении заявки файлы не удаляются (архив), но помечаются как неактивные.

### 7. Обработка ошибок и безопасность

- Все внешние запросы должны иметь таймауты (10 сек) и обработку исключений.
- При ошибке загрузки фото — предложить повторить или продолжить без фото.
- Ограничение на размер загружаемого файла (макс. 10 МБ).
- Валидация дат (нельзя записаться в прошлое).
- Рассылка должна иметь искусственную задержку 0.05 сек между сообщениями.
- Обработка `TelegramForbiddenError` — удалять пользователя из активной рассылки.

### 8. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
```

**systemd сервис:**
```ini
[Unit]
Description=Hair Wishlist Bot
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/hair-wishlist-bot
ExecStart=/usr/bin/python3 main.py
Restart=always

[Install]
WantedBy=multi-user.target
```

**Инструкция по развёртыванию на VPS:**

1. Скопировать файлы проекта на сервер
2. Создать файл `.env` с необходимыми переменными
3. Запустить через Docker: `docker-compose up -d`
4. Или через systemd:
   ```bash
   sudo cp hair-wishlist-bot.service /etc/systemd/system/
   sudo systemctl daemon-reload
   sudo systemctl enable hair-wishlist-bot
   sudo systemctl start hair-wishlist-bot
   ```

5. Проверить логи: `journalctl -u hair-wishlist-bot -f`
