# Промт для создания Telegram-бота «Скидка на повторный прием»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Скидка на повторный прием».

**Контекст проекта:**
Этот бот продаётся частным клиникам, косметологам, стоматологам и другим медицинским сервисам. Цель — автоматизировать возврат пациентов: бот отслеживает дату первого визита и через заданный период (например, 3 месяца) автоматически высылает персональный промокод на скидку. Это повышает лояльность и заполняет расписание.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение с названием клиники.
   - Под сообщением — клавиатура (InlineKeyboardMarkup):
     - 🎁 Мои скидки
     - 📅 История визитов
     - 📞 Записаться
     - ℹ️ О клинике

2. **Обработка кнопок и сценарии:**
   - **Мои скидки:** Показывает активные промокоды пользователя с сроком действия. Если промокод есть — отображает код и кнопку «Применить» (ссылка на запись).
   - **История визитов:** Список дат посещений клиники.
   - **Записаться:** Кнопка-ссылка на внешний сервис записи или телефон.
   - **О клинике:** Адрес, режим работы, услуги.

3. **Администрирование:**
   - Команда `/add_visit [user_id] [date]` — добавить запись о визите пациента. Бот автоматически запланирует отправку промокода через N месяцев.
   - Команда `/list_patients` — показать всех пациентов с датами последних визитов.
   - Команда `/send_promo [user_id]` — вручную отправить промокод пациенту.
   - Команда `/set_promo_rules [months] [discount_percent]` — настроить правила выдачи промокодов.
   - Команда `/stats` — статистика: сколько промокодов выдано, использовано, конверсия.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv, aiosqlite, asyncio, apscheduler (для отложенных отправок).
   - **Структура проекта:**
```
/bot
├── main.py
├── config.py
├── handlers/
│   ├── patient.py
│   └── admin.py
├── keyboards/
│   └── main_menu.py
├── scheduler.py # планировщик отправок
├── database.py
├── .env
└── requirements.txt
```
   - **Конфигурация (.env):**
```
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
CLINIC_NAME=«Дентал+»
CLINIC_PHONE=+79001234567
CLINIC_ADDRESS=г. Москва, ул. Ленина, 10

PROMO_DISCOUNT_PERCENT=15
PROMO_VALIDITY_DAYS=30

```
   - **Хранение данных:** SQLite с таблицами:
     - `visits` (id, user_id, visit_date, created_at)
     - `promocodes` (id, user_id, code, discount_percent, issued_at, expires_at, is_used, used_at)
     - `users` (user_id, username, first_name, last_seen)

5. **Обработка ошибок и безопасность:**
   - Промокоды — уникальные строки 8–10 символов.
   - Автоматическая проверка срока действия промокода перед использованием.
   - Логирование всех выданных и использованных промокодов.
   - Планировщик проверяет базу раз в сутки и отправляет промокоды.

6. **Деплой:**
   - Код готов к запуску через Docker или systemd.
   - Включить Dockerfile, docker-compose.yml, README.md.
