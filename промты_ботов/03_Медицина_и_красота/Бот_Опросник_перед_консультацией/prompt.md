# Промт для создания Telegram-бота «Опросник перед консультацией»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Опросник перед консультацией».

**Контекст проекта:**
Этот бот продаётся психологам, диетологам, косметологам и другим специалистам, проводящим консультации. Цель — собрать анамнез до приёма: пациент отвечает на серию вопросов, врач получает структурированную сводку. Это экономит 15–20 минут приёма и позволяет врачу лучше подготовиться.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение с объяснением цели опроса.
   - Под сообщением — клавиатура (InlineKeyboardMarkup):
     - 📝 Пройти опросник
     - 📋 Мои ответы
     - 📅 Записаться на консультацию
     - ℹ️ О специалисте
     - 📞 Контакты

2. **Обработка кнопок и сценарии:**
   - **Пройти опросник:** Бот задаёт вопросы по очереди (5–10 вопросов в зависимости от специализации). Вопросы загружаются из файла `questionnaires.json`. Типы вопросов: текст, выбор из вариантов, шкала 1–10. После завершения бот формирует сводку и отправляет её врачу.
   - **Мои ответы:** Показывает историю пройденных опросников с датами.
   - **Записаться на консультацию:** Кнопка-ссылка на внешний сервис записи или телефон.
   - **О специалисте:** Информация о враче (образование, опыт, специализация).
   - **Контакты:** Телефон, email, адрес.

3. **Администрирование (врач):**
   - Команда `/get_answers [user_id]` — получить ответы конкретного пациента.
   - Команда `/list_clients` — показать всех пациентов, прошедших опрос.
   - Команда `/export_answers [user_id]` — выгрузить ответы в формате PDF или TXT.
   - Команда `/update_questions [specialization]` — загрузить новый набор вопросов (через файл).
   - Команда `/broadcast [текст]` — рассылка всем пациентам.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv, aiosqlite, asyncio, aiofiles, reportlab (для PDF).
   - **Структура проекта:**
```
/bot
├── main.py
├── config.py
├── handlers/
│   ├── client.py
│   └── doctor.py
├── keyboards/
│   └── main_menu.py
├── data/
│   └── questionnaires.json
├── database.py
├── .env
└── requirements.txt
```
   - **Конфигурация (.env):**
```
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
SPECIALIST_NAME=Анна Иванова
SPECIALIST_TITLE=Клинический психолог
SPECIALIST_EXPERIENCE=10 лет
BOOKING_URL=https://booking.ru/...

```
   - **Хранение данных:** SQLite с таблицами:
     - `questionnaires` (id, user_id, specialization, questions_json, created_at)
     - `answers` (id, questionnaire_id, user_id, answers_json, submitted_at)
     - `users` (user_id, username, first_name, last_seen)

5. **Обработка ошибок и безопасность:**
   - Сохранение прогресса опроса (если пользователь прервал, можно продолжить).
   - Конфиденциальность данных (доступ только у врача).
   - Экспорт данных в защищённом формате.

6. **Деплой:**
   - Код готов к запуску через Docker или systemd.
   - Включить Dockerfile, docker-compose.yml, README.md.
