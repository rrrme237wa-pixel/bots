# Промт для создания Telegram-бота «Оцени сервис»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Оцени сервис».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для салонов красоты, барбершопов, спа-центров и других заведений сферы услуг. Цель — автоматически собирать отзывы и оценки от клиентов после визита, чтобы владелец мог отслеживать качество сервиса и быстро реагировать на негатив.

**Монетизация:** Продажа готового решения салонам за 5000–8000 рублей, либо ежемесячная подписка 900 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение от имени салона.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - ⭐ Оставить оценку
  - 📊 Моя история отзывов
  - 📞 Контакты
  - ℹ️ Как это работает

### 2. Обработка кнопок

#### Оставить оценку
- Бот запрашивает номер заказа/визита (или клиент выбирает из последних посещений).
- После подтверждения бот предлагает оценить сервис по шкале от 1 до 5 звёзд.
- При оценке 1–3 звезды:
  - Бот просит описать проблему в текстовом поле.
  - Отправляет жалобу управляющему в ЛС немедленно.
- При оценке 4–5 звёзд:
  - Бот благодарит клиента.
  - Предлагает поделиться отзывом в соцсетях (кнопки-ссылки).
  - Просит разрешение на публикацию отзыва на сайте салона.

#### Моя история отзывов
- Отображение всех оставленных клиентом оценок.
- Для каждой оценки: дата, количество звёзд, текст отзыва, статус обработки.

#### Контакты
- Телефон салона, адрес, ссылки на соцсети.

#### Как это работает
- Инструкция: «Посетите салон → Получите уведомление → Оставьте оценку → Помогите нам стать лучше».

### 3. Администрирование (для владельца салона)

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — статистика:
  - Средняя оценка за сегодня/неделю/месяц
  - Количество отзывов по звёздам (распределение)
  - Количество нерешённых жалоб

- `/reviews` — просмотр последних отзывов с фильтрацией по оценке.

- `/alert_negative` — список всех жалоб (оценки 1–3) с контактами клиентов.

- `/export_reviews` — выгрузка всех отзывов в CSV/Excel.

- `/broadcast [текст]` — рассылка клиентам.

- `/response [review_id] [текст]` — отправить ответ на отзыв.

### 4. Технические детали

**Стек:**
- Python 3.10+
- aiogram 3.x
- python-dotenv для конфигурации
- aiosqlite для хранения данных
- asyncio

**Структура проекта:**
```

├── config.py                  # загрузка настроек из .env
```

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `phone` TEXT
- `first_seen` TIMESTAMP
- `last_seen` TIMESTAMP

Таблица `visits`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER
- `visit_date` DATETIME
- `service_type` TEXT
- `master_name` TEXT

Таблица `reviews`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER
- `visit_id` INTEGER
- `rating` INTEGER (1–5)
- `comment` TEXT
- `is_public` BOOLEAN DEFAULT 0
- `status` TEXT (new/in_resolved/resolved)
- `admin_response` TEXT
- `created_at` TIMESTAMP

При каждом `/start` обновлять запись в `users`.

### 6. Обработка ошибок и безопасность

- Таймауты на запросы (10 сек).
- Анонимность: при отправке жалобы мастеру имя клиента скрывается (только контакты для связи).
- Рассылка с задержкой 0.05 сек.
- Обработка `TelegramForbiddenError`.

### 7. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
```

**Инструкция по развёртыванию:**
1. Скопировать файлы на сервер
2. Создать `.env`
3. `docker-compose up -d`
