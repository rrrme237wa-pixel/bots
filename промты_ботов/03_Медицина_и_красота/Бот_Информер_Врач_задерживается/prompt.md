Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного Telegram-бота «Информер: Врач задерживается».

**Контекст проекта:**
Бот для клиник и медицинских центров. Позволяет администратору быстро уведомить всех пациентов о задержке врача. Отличается от аналогичных ботов упрощенным интерфейсом и фокусом на скорость оповещения. Монетизация: продажа как готовое решение для регистратур.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При `/start` бот определяет роль (пациент или админ).
   - Меню пациента:
     - 📅 Записаться на прием
     - 🔔 Статус приема
     - 📞 Контакты
   - Меню админа:
     - ⚡ Объявить задержку
     - ✅ Отбой тревоги
     - 👥 Пациенты в очереди

2. **Обработка кнопок:**
   - **Объявить задержку (админ):**
     - Быстрый выбор времени задержки: 15, 30, 45, 60 мин.
     - Опционально: причина задержки.
     - Мгновенная рассылка всем пациентам с записью на ближайшие 3 часа.
   - **Отбой тревоги:** Снимает статус задержки, пациенты получают уведомление "Врач вернулся к работе".
   - **Статус приема (пациент):** Показывает текущий статус (по расписанию / задержка X мин).

3. **Администрирование:**
   - Команда `/queue` — показать текущую очередь пациентов.
   - Команда `/history` — история всех задержек за период.
   - Настройка шаблонов сообщений через JSON.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, sqlite3, python-dotenv.
   - **Структура проекта:**
/bot
├── main.py
├── config.py
├── handlers/
│ ├── patient.py
│ └── admin.py
├── keyboards/
│ └── menus.py
├── data/
│ └── templates.json
├── .env
└── requirements.txt
   - **Конфигурация (.env):**
BOT_TOKEN=...
ADMIN_IDS=123456789
CLINIC_NAME="МедЦентр"
CLINIC_PHONE="+79990000000"

5. **Хранение данных:**
   - SQLite таблицы:
     - `users` (user_id, role, first_name, phone)
     - `appointments` (id, user_id, appointment_time, status)
     - `delays` (id, start_time, duration_min, reason, resolved)

6. **Обработка ошибок и безопасность:**
   - Рассылка с защитой от блокировок (задержка между сообщениями).
   - Логирование всех действий админа.
   - Автоматический сброс статуса задержки через указанное время.

7. **Деплой:**
   - Dockerfile, docker-compose.yml.
   - Инструкция по развертыванию.

**Формат ответа:**
Предоставь полный код всех файлов проекта. В конце дай инструкцию по развёртыванию.
