# Промт для создания Telegram-бота «Книга жалоб онлайн»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Книга жалоб онлайн».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для кафе, ресторанов, отелей, магазинов и других заведений сферы услуг. Цель — предоставить гостям анонимный канал для отправки жалоб и предложений управляющему, чтобы быстро решать проблемы и улучшать сервис. Все жалобы фиксируются с датой для отчётности.

**Монетизация:** Продажа готового решения заведениям за 4000–6000 рублей, либо ежемесячная подписка 700 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение от имени заведения.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - 📝 Оставить жалобу
  - 💡 Предложение
  - 📊 Статус моей заявки
  - ℹ️ Как это работает

### 2. Обработка кнопок

#### Оставить жалобу
- Бот запускает FSM для пошагового сбора информации:
  1. **Тип проблемы:** Выбор из списка (грязь, обслуживание, еда, шум, другое).
  2. **Дата и время визита:** Выбор из календаря.
  3. **Описание проблемы:** Текстовое поле (минимум 20 символов).
  4. **Фото/видео доказательство:** Опционально, можно загрузить до 5 файлов.
  5. **Контакты для связи:** Телефон или email (опционально, для анонимности).
  6. **Анонимность:** Выбор — показать имя управляющему или скрыть.
- После заполнения бот показывает итог для подтверждения.
- При подтверждении заявка сохраняется, управляющий получает уведомление в ЛС.
- Клиент получает номер заявки для отслеживания.

#### Предложение
- Аналогично жалобе, но без негативной окраски.
- Поле «Идея по улучшению».

#### Статус моей заявки
- Клиент вводит номер заявки.
- Бот показывает статус (новая/в работе/решена) и комментарий управляющего.

### 3. Администрирование

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — статистика:
  - Количество жалоб за период
  - Среднее время решения
  - Категории проблем (диаграмма)

- `/complaints` — список всех жалоб с фильтрацией по статусу.

- `/update_status [id] [status]` — изменить статус заявки.

- `/respond [id] [текст]` — отправить ответ клиенту.

- `/export` — выгрузка отчёта в CSV/Excel.

### 4. Технические детали

**Стек:**
- Python 3.10+
- aiogram 3.x
- python-dotenv для конфигурации
- aiosqlite для хранения данных
- aiofiles для работы с файлами
- asyncio

**Структура проекта:**
```
/bot
├── main.py                    # точка входа
├── config.py                  # загрузка настроек из .env
├── handlers/
│   ├── user.py                # обработчики пользовательских команд
│   ├── admin.py               # обработчики админских команд
│   └── complaints.py          # логика жалоб (FSM)
├── keyboards/
│   └── main_menu.py           # главное меню
├── utils/
│   ├── database.py            # работа с SQLite
│   └── media_handler.py       # обработка файлов
├── data/
│   ├── uploads/               # загруженные файлы
│   └── database.db            # SQLite база данных
├── .env                       # токен бота, ID админа и т.д.
├── requirements.txt
└── README.md                  # инструкция по запуску
```

**Конфигурация (.env):**
```env
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
RESTAURANT_NAME=Ресторан «Премиум»
ANONYMOUS_MODE=True
TIMEZONE=Europe/Moscow
```

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `first_seen` TIMESTAMP

Таблица `complaints`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER
- `complaint_number` TEXT UNIQUE
- `type` TEXT
- `visit_date` DATETIME
- `description` TEXT
- `is_anonymous` BOOLEAN DEFAULT 0
- `status` TEXT (new/in_progress/resolved/closed)
- `admin_response` TEXT
- `created_at` TIMESTAMP
- `resolved_at` TIMESTAMP

Таблица `complaint_files`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `complaint_id` INTEGER
- `file_path` TEXT
- `file_type` TEXT

При каждом `/start` обновлять запись в `users`.

### 6. Обработка ошибок и безопасность

- Таймауты на запросы (10 сек).
- Анонимность: при `is_anonymous = 1` имя клиента скрывается.
- Валидация минимальной длины описания.
- Рассылка с задержкой 0.05 сек.

### 7. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
```

**Инструкция по развёртыванию:**
1. Скопировать файлы на сервер
2. Создать `.env`
3. `docker-compose up -d`
