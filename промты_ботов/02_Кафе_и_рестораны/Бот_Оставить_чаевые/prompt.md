# Промт для создания Telegram-бота «Оставить чаевые»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Оставить чаевые».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для кафе, ресторанов, баров, отелей и других заведений сферы услуг. Цель — предоставить гостям удобный способ оставить чаевые конкретному официанту, бариста или сотруднику через QR-код или ссылку, без необходимости иметь наличные.

**Монетизация:** Продажа готового решения заведениям за 3000–5000 рублей, либо ежемесячная подписка 500 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение от имени заведения.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - 💰 Оставить чаевые
  - 👤 Выбрать сотрудника
  - 📊 Моя история
  - ℹ️ Как это работает

### 2. Обработка кнопок

#### Оставить чаевые
- Бот показывает QR-код или ссылку для быстрого перехода.
- При сканировании QR-кода на столике — автоматически определяется номер столика и официальный.
- Клиент выбирает сумму чаевых (быстрые кнопки: 50₽, 100₽, 200₽, 500₽ или своя сумма).
- Выбор способа оплаты (перевод на карту, СБП).
- После оплаты — подтверждение и благодарность.

#### Выбрать сотрудника
- Список сотрудников с фото и именами.
- Выбор конкретного человека.
- Ввод суммы и отправка.

#### Моя история
- История всех отправленных чаевых.
- Для каждой транзакции: дата, сумма, сотрудник, статус.

### 3. Администрирование

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — статистика:
  - Общая сумма чаевых за период
  - Количество транзакций
  - Топ сотрудников по чаевым

- `/employees` — управление сотрудниками:
  - Добавить сотрудника (ФИО, фото, реквизиты)
  - Удалить сотрудника
  - Редактировать данные

- `/export` — выгрузка отчёта в CSV/Excel.

- `/withdraw [employee_id]` — запрос на вывод средств сотруднику.

### 4. Технические детали

**Стек:**
- Python 3.10+
- aiogram 3.x
- python-dotenv для конфигурации
- aiosqlite для хранения данных
- qrcode для генерации QR
- asyncio

**Структура проекта:**
```
/bot
├── main.py                    # точка входа
├── config.py                  # загрузка настроек из .env
├── handlers/
│   ├── user.py                # обработчики пользовательских команд
│   ├── admin.py               # обработчики админских команд
│   └── tips.py                # логика чаевых
├── keyboards/
│   └── main_menu.py           # главное меню
├── utils/
│   ├── database.py            # работа с SQLite
│   └── qr_generator.py        # генерация QR
├── data/
│   └── database.db            # SQLite база данных
├── .env                       # токен бота, ID админа и т.д.

```

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `first_seen` TIMESTAMP

Таблица `employees`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `name` TEXT
- `photo_path` TEXT
- `payment_details` TEXT
- `is_active` BOOLEAN DEFAULT 1

Таблица `tips`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER
- `employee_id` INTEGER
- `amount` INTEGER
- `table_number` TEXT
- `status` TEXT (pending/completed/withdrawn)
- `created_at` TIMESTAMP

При каждом `/start` обновлять запись в `users`.

### 6. Обработка ошибок и безопасность

- Таймауты на запросы (10 сек).
- Валидация сумм (минимум 10₽).
- Логирование всех транзакций.

### 7. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
```

**Инструкция по развёртыванию:**
1. Скопировать файлы на сервер
2. Создать `.env`
3. `docker-compose up -d`
