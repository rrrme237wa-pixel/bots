# Промт для создания Telegram-бота «Стоп-лист»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Стоп-лист».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для кафе, ресторанов, столовых и других заведений общественного питания. Цель — позволить официантам и поварам быстро обновлять статус блюд в меню (закончилось/в наличии), чтобы гости видели актуальную информацию без необходимости звонить на кухню или администратору.

**Монетизация:** Продажа готового решения заведениям за 5000–8000 рублей, либо ежемесячная подписка 1000 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

Бот имеет два режима работы: **для гостей** и **для персонала**.

#### Для гостей:
- При команде `/start` бот отправляет приветственное сообщение от имени заведения.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - 📋 Меню
  - 🍽️ Сегодня в наличии
  - 🔴 Стоп-лист
  - ℹ️ О заведении

#### Для персонала:
- Авторизация по коду доступа (задаётся в `.env`).
- После авторизации — меню управления:
  - 📝 Обновить статус блюда
  - 📊 Текущий стоп-лист
  - ➕ Добавить новое блюдо
  - 📈 Статистика

### 2. Обработка кнопок

#### Для гостей:

**Меню:**
- Бот показывает полное меню с категориями (Закуски, Салаты, Горячее, Напитки и т.д.).
- Каждое блюдо сопровождается фото, описанием, ценой и статусом:
  - ✅ В наличии
  - 🔴 Закончилось
  - 🕒 Ожидается через N минут

**Сегодня в наличии:**
- Фильтр меню, показывающий только доступные блюда.

**Стоп-лист:**
- Список блюд, которые временно недоступны.
- Для каждого блюда указано ожидаемое время появления (если известно).

#### Для персонала:

**Обновить статус блюда:**
- Выбор категории → выбор блюда.
- Установка статуса: «В наличии» / «Закончилось» / «Ожидается».
- Если «Ожидается» — ввод времени готовности (например, «через 30 минут»).
- После обновления бот может отправить уведомление подписчикам (опционально).

**Текущий стоп-лист:**
- Просмотр всех блюд в статусе «Закончилось».
- Возможность быстро вернуть статус «В наличии».

**Добавить новое блюдо:**
- Загрузка фото.
- Ввод названия, описания, цены, категории.
- Добавление в меню.

### 3. Администрирование

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — статистика:
  - Количество просмотров меню за день
  - Популярные блюда
  - Количество обновлений стоп-листа за день

- `/staff` — управление персоналом:
  - Добавить сотрудника (выдача кода доступа)
  - Удалить сотрудника
  - Список активных сотрудников

- `/export_menu` — выгрузка меню в PDF/CSV.

- `/broadcast [текст]` — рассылка гостям (например, об изменении меню).

### 4. Технические детали

**Стек:**
- Python 3.10+
- aiogram 3.x
- python-dotenv для конфигурации
- aiosqlite для хранения данных
- aiofiles для работы с файлами
- asyncio

**Структура проекта:**
```
/bot
├── main.py                    # точка входа
├── config.py                  # загрузка настроек из .env
├── handlers/
│   ├── guest.py               # обработчики для гостей
│   ├── staff.py               # обработчики для персонала
│   └── admin.py               # обработчики админских команд
├── keyboards/
│   ├── guest_menu.py          # меню для гостей
│   └── staff_menu.py          # меню для персонала
├── utils/
│   ├── database.py            # работа с SQLite
│   └── menu_manager.py        # логика управления меню
├── data/
│   ├── menu_photos/           # фото блюд
│   └── database.db            # SQLite база данных
├── .env                       # токен бота, ID админа и т.д.
├── requirements.txt
└── README.md                  # инструкция по запуску
```

**Конфигурация (.env):**
```env

STAFF_ACCESS_CODE=1234
```

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `role` TEXT (guest/staff/admin)
- `first_seen` TIMESTAMP

Таблица `categories`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `name` TEXT
- `order` INTEGER

Таблица `dishes`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `category_id` INTEGER
- `name` TEXT
- `description` TEXT
- `price` INTEGER
- `photo_path` TEXT
- `status` TEXT (available/unavailable/expected)
- `expected_time` DATETIME
- `updated_at` TIMESTAMP
- `updated_by` INTEGER

Таблица `staff`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER UNIQUE
- `access_code` TEXT
- `is_active` BOOLEAN DEFAULT 1
- `added_at` TIMESTAMP

При каждом `/start` обновлять запись в `users`.

### 6. Обработка ошибок и безопасность

- Таймауты на запросы (10 сек).
- Защита от несанкционированного доступа (проверка кода персонала).
- Логирование всех изменений меню (кто, когда, какое блюдо изменил).
- Рассылка с задержкой 0.05 сек.

### 7. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
      - ./menu_photos:/app/menu_photos
```

**Инструкция по развёртыванию:**
1. Скопировать файлы на сервер
2. Создать `.env`
3. `docker-compose up -d`
