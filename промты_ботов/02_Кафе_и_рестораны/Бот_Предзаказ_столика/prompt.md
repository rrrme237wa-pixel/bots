# Промт для создания Telegram-бота «Предзаказ столика»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Предзаказ столика».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для кафе, ресторанов, баров и других заведений общественного питания. Цель — позволить гостям забронировать столик без звонка, а администратору — получать заявки в удобном формате без интеграции со сложной CRM-системой.

**Монетизация:** Продажа готового решения заведениям за 4000–7000 рублей, либо ежемесячная подписка 800 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение от имени заведения.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - 🪑 Забронировать столик
  - 📋 Мои бронирования
  - 📞 Контакты
  - ℹ️ О заведении

### 2. Обработка кнопок

#### Забронировать столик
- Бот запускает FSM для пошагового сбора информации:
  1. **Дата:** Выбор даты из календаря (на сегодня и до 30 дней вперёд).
  2. **Время:** Выбор времени с шагом 30 минут.
  3. **Количество гостей:** 1–20 человек.
  4. **Зона посадки:** Выбор из вариантов (основной зал, VIP-зона, летняя терраса, у бара).
  5. **Контактные данные:** Телефон (автоматически или вручную), имя.
  6. **Особые пожелания:** Текстовое поле (день рождения, аллергия и т.п.).
- После заполнения бот показывает итоговую информацию для подтверждения.
- При подтверждении заявка сохраняется и отправляется администратору.
- Клиент получает номер бронирования.

#### Мои бронирования
- Список активных и прошедших бронирований клиента.
- Для каждого: дата, время, статус (подтверждено/отменено/завершено).
- Кнопка «Отменить бронь» для активных заявок.

#### Контакты
- Телефон, адрес, режим работы.
- Кнопка «Проложить маршрут».

#### О заведении
- Описание, фото интерьера, меню (опционально).

### 3. Администрирование

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — статистика:
  - Количество бронирований на сегодня/неделю
  - Среднее количество гостей
  - Популярные дни и время

- `/reservations` — просмотр всех заявок на сегодня/завтра.

- `/confirm [id]` — подтвердить бронь.

- `/cancel [id]` — отменить бронь (с уведомлением клиента).

- `/calendar` — показать календарь бронирований на неделю.

- `/export` — выгрузка бронирований в CSV.

### 4. Технические детали

**Стек:**
- Python 3.10+
- aiogram 3.x
- python-dotenv для конфигурации
- aiosqlite для хранения данных
- asyncio

**Структура проекта:**
```
/bot
├── main.py                    # точка входа
├── config.py                  # загрузка настроек из .env
├── handlers/
│   ├── user.py                # обработчики пользовательских команд
│   ├── admin.py               # обработчики админских команд
│   └── booking.py             # логика бронирования (FSM)
├── keyboards/
│   ├── main_menu.py           # главное меню
│   └── booking_kb.py          # клавиатуры бронирования
├── utils/

│   └── calendar.py            # календарь для выбора даты
├── data/

```

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `phone` TEXT
- `first_seen` TIMESTAMP

Таблица `reservations`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER
- `reservation_date` DATE
- `reservation_time` TIME
- `guests_count` INTEGER
- `zone` TEXT
- `special_requests` TEXT
- `status` TEXT (pending/confirmed/cancelled/completed)
- `booking_number` TEXT UNIQUE
- `created_at` TIMESTAMP

При каждом `/start` обновлять запись в `users`.

### 6. Обработка ошибок и безопасность

- Таймауты на запросы (10 сек).
- Проверка на двойное бронирование одного времени.
- Автоматическое удаление «забытых» броней через 24 часа после времени брони.
- Рассылка с задержкой 0.05 сек.

### 7. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
```

**Инструкция по развёртыванию:**
1. Скопировать файлы на сервер
2. Создать `.env`
3. `docker-compose up -d`
