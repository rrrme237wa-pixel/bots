# Промт для создания Telegram-бота «Комплимент от шефа»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Комплимент от шефа».

## Контекст проекта

Этот бот продаётся как готовое цифровое решение для кафе, ресторанов и баров. Цель — генерировать уникальные QR-коды на бесплатный десерт или напиток для гостей, которые действуют один раз и списываются при сканировании официантом. Это повышает лояльность клиентов и стимулирует повторные визиты.

**Монетизация:** Продажа готового решения заведениям за 4000–6000 рублей, либо ежемесячная подписка 700 рублей за поддержку и обновления.

## Функциональные требования

### 1. Стартовое сообщение и меню

- При команде `/start` бот отправляет приветственное сообщение от имени заведения.
- Под сообщением — клавиатура InlineKeyboardMarkup:
  - 🎁 Получить комплимент
  - 📱 Мои сертификаты
  - 📞 Контакты
  - ℹ️ Условия акции

### 2. Обработка кнопок

#### Получить комплимент
- Бот проверяет, есть ли у клиента активный сертификат.
- Если нет — генерирует уникальный QR-код с токеном.
- QR-код сохраняется в базе данных со статусом «активен».
- Клиент получает изображение с QR-кодом и инструкцию:
  ```
  Ваш комплимент от шефа! 🎉
  
  Покажите этот QR-код официанту до оплаты счёта.
  Действует один раз.
  
  Срок действия: 30 дней
  ```

#### Мои сертификаты
- Список всех полученных клиентом сертификатов.
- Для каждого: статус (активен/использован/истёк), дата получения, срок действия.
- Активный сертификат показывается с большим QR-кодом.

#### Контакты
- Телефон, адрес, режим работы.

#### Условия акции
- Правила использования комплиментов.
- Срок действия, ограничения.

### 3. Администрирование

Для владельца бота (ID админа задаётся в `.env`) доступны команды:

- `/admin` — статистика:
  - Количество выданных сертификатов
  - Количество использованных
  - Процент конверсии

- `/validate [token]` — проверить и списать сертификат (для официантов).

- `/certificates` — список всех сертификатов.

- `/export` — выгрузка в CSV.

### 4. Технические детали

**Стек:**
- Python 3.10+
- aiogram 3.x
- python-dotenv для конфигурации
- aiosqlite для хранения данных
- qrcode для генерации QR-кодов
- asyncio

**Структура проекта:**
```
/bot
├── main.py                    # точка входа
├── config.py                  # загрузка настроек из .env
├── handlers/
│   ├── user.py                # обработчики пользовательских команд
│   ├── admin.py               # обработчики админских команд
│   └── qr.py                  # логика QR-кодов
├── keyboards/
│   └── main_menu.py           # главное меню
├── utils/
│   ├── database.py            # работа с SQLite
│   └── qr_generator.py        # генерация QR
├── data/
│   ├── qr_codes/              # сгенерированные QR
│   └── database.db            # SQLite база данных
├── .env                       # токен бота, ID админа и т.д.
├── requirements.txt
└── README.md                  # инструкция по запуску
```

**Конфигурация (.env):**
```env
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
RESTAURANT_NAME=Ресторан «Gusto»
CERTIFICATE_VALIDITY_DAYS=30
COMPLIMENT_TEXT=Бесплатный десерт в подарок!
TIMEZONE=Europe/Moscow
```

### 5. Хранение данных

**SQLite база данных:**

Таблица `users`:
- `user_id` INTEGER PRIMARY KEY
- `username` TEXT
- `first_name` TEXT
- `first_seen` TIMESTAMP

Таблица `certificates`:
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `user_id` INTEGER
- `token` TEXT UNIQUE
- `qr_path` TEXT
- `status` TEXT (active/used/expired)
- `issued_at` TIMESTAMP
- `expires_at` DATETIME
- `used_at` TIMESTAMP
- `used_by_staff_id` INTEGER

При каждом `/start` обновлять запись в `users`.

### 6. Обработка ошибок и безопасность

- Таймауты на запросы (10 сек).
- Уникальность токенов (генерация через uuid).
- Защита от повторного использования.
- Рассылка с задержкой 0.05 сек.

### 7. Деплой

**Dockerfile:**
```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./data:/app/data
```

**Инструкция по развёртыванию:**
1. Скопировать файлы на сервер
2. Создать `.env`
3. `docker-compose up -d`
