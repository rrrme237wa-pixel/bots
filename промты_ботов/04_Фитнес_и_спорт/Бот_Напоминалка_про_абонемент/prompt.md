# Промт для создания Telegram-бота «Напоминалка про абонемент»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Напоминалка про абонемент».

**Контекст проекта:**
Этот бот продаётся фитнес-клубам, студиям йоги, танцевальным школам и другим сервисам с абонементами. Цель — автоматически уведомлять клиентов об окончании срока действия абонемента (за 7 и 3 дня), чтобы повысить продления и снизить отток. Бот интегрируется с базой клиентов клуба.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение.
   - Под сообщением — клавиатура (InlineKeyboardMarkup):
     - 📅 Статус абонемента
     - 💳 Продлить абонемент
     - 📊 История посещений
     - 📞 Контакты клуба

2. **Обработка кнопок и сценарии:**
   - **Статус абонемента:** Показывает тип абонемента, дату начала, дату окончания, количество оставшихся дней и визитов.
   - **Продлить абонемент:** Кнопка-ссылка на оплату или форму заявки.
   - **История посещений:** Список дат посещений клуба.
   - **Контакты клуба:** Адрес, телефон, режим работы.

3. **Администрирование:**
   - Команда `/add_subscription [user_id] [type] [start_date] [end_date] [visits_left]` — добавить абонемент клиенту.
   - Команда `/list_expiring` — показать клиентов, у которых абонемент истекает в ближайшие 7 дней.
   - Команда `/extend [user_id] [days]` — продлить абонемент.
   - Команда `/broadcast [текст]` — рассылка всем клиентам.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv, aiosqlite, asyncio, apscheduler.
   - **Структура проекта:**
```
/bot
├── main.py
├── config.py
├── handlers/
│   ├── client.py
│   └── admin.py
├── keyboards/
│   └── main_menu.py
├── scheduler.py
├── database.py
├── .env
└── requirements.txt
```
   - **Конфигурация (.env):**
```
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
CLUB_NAME=FitClub
CLUB_PHONE=+79001234567
REMINDER_DAYS=7,3
TIMEZONE=Europe/Moscow
```
   - **Хранение данных:** SQLite с таблицами:
     - `subscriptions` (id, user_id, type, start_date, end_date, visits_total, visits_left, is_active)
     - `visits` (id, user_id, visit_date)
     - `users` (user_id, username, first_name, phone, last_seen)

5. **Обработка ошибок и безопасность:**
   - Планировщик проверяет базу раз в сутки.
   - Уведомления отправляются в 10:00 по местному времени.
   - Логирование всех напоминаний.

6. **Деплой:**
   - Код готов к запуску через Docker или systemd.
   - Включить Dockerfile, docker-compose.yml, README.md.
