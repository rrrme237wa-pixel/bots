# Промт для создания Telegram-бота «Количество мест в группе»

Ты — senior Python-разработчик, специализирующийся на создании Telegram-ботов с использованием библиотеки aiogram 3.x. Твоя задача — написать полноценного, готового к развёртыванию на сервере Telegram-бота «Количество мест в группе».

**Контекст проекта:**
Этот бот продаётся студиям йоги, пилатеса, фитнес-клубам и танцевальным школам. Цель — показывать клиентам свободные места в группах на занятиях и позволять записываться без звонков администратору. Бот снижает нагрузку на ресепшн и повышает конверсию записей.

**Функциональные требования:**

1. **Стартовое сообщение и меню:**
   - При команде `/start` бот отправляет приветственное сообщение с названием студии.
   - Под сообщением — клавиатура (InlineKeyboardMarkup):
     - 📅 Расписание занятий
     - 🧘 Мои записи
     - 👤 Личный кабинет
     - 📞 Контакты

2. **Обработка кнопок и сценарии:**
   - **Расписание занятий:** Бот показывает список занятий на ближайшие 7 дней. Каждое занятие — кнопка с названием, временем, именем тренера и количеством свободных мест (например, «Йога 18:00 | Анна | 3/10 мест»). При выборе — детальная информация и кнопка «Записаться» (если есть места).
   - **Мои записи:** Показывает предстоящие занятия пользователя с возможностью отмены записи.
   - **Личный кабинет:** Имя, телефон, количество посещений, абонемент (если есть).
   - **Контакты:** Адрес студии, телефон, кнопка «Позвонить», геолокация.

3. **Администрирование:**
   - Команда `/add_class [name] [trainer] [datetime] [max_slots]` — добавить занятие в расписание.
   - Команда `/list_classes` — показать все занятия на неделю.
   - Команда `/cancel_class [class_id]` — отменить занятие с уведомлением записанных клиентов.
   - Команда `/book [user_id] [class_id]` — записать клиента на занятие (для админа).
   - Команда `/broadcast [текст]` — рассылка всем подписчикам.

4. **Технические детали:**
   - **Стек:** Python 3.10+, aiogram 3.x, python-dotenv, aiosqlite, asyncio, datetime.
   - **Структура проекта:**
```
/bot
├── main.py
├── config.py
├── handlers/
│   ├── client.py
│   └── admin.py
├── keyboards/
│   └── main_menu.py
├── database.py
├── .env
└── requirements.txt
```
   - **Конфигурация (.env):**
```
BOT_TOKEN=123456:ABCdef...
ADMIN_ID=123456789
STUDIO_NAME=«Йога-студия»
STUDIO_ADDRESS=г. Москва, ул. Мира, 15
STUDIO_PHONE=+79001234567
TIMEZONE=Europe/Moscow
```
   - **Хранение данных:** SQLite с таблицами:
     - `classes` (id, name, trainer, datetime, max_slots, booked_slots, is_active, created_at)
     - `bookings` (id, class_id, user_id, username, booked_at, status)
     - `users` (user_id, username, first_name, phone, last_seen)

5. **Обработка ошибок и безопасность:**
   - Проверка на дублирующую запись (один пользователь — одно место в группе).
   - Уведомление при отмене занятия всем записанным.
   - Автоматическое обновление счётчика мест после записи/отмены.

6. **Деплой:**
   - Код готов к запуску через Docker или systemd.
   - Включить Dockerfile, docker-compose.yml, README.md.
